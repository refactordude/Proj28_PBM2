<objective>
Create `app_v2/services/cache.py`, the v2.0 replacement for `@st.cache_data`. It provides `cachetools.TTLCache` + `threading.Lock()` wrappers for the three framework-agnostic `_core()` functions added in plan 01-01 (`list_platforms_core`, `list_parameters_core`, `fetch_cells_core`). Phase 2 onward imports the wrapper names from this module; the lower-level `_core()` functions are NEVER called directly by routes.

Why thread safety is non-negotiable (Pitfall 11):
- FastAPI `def` routes run concurrently in a threadpool (INFRA-05 convention from plan 01-03).
- `cachetools.TTLCache` is NOT thread-safe — concurrent access can raise `RuntimeError: dictionary changed size during iteration` during eviction, or return partially-written entries.
- Every `TTLCache` in this module is paired with a module-level `threading.Lock()` — enforced from the first write so no route ever sees an unprotected cache.

Why keys must exclude the adapter (Pitfall 11 — second sub-trap):
- The `DBAdapter` object is NOT hashable by default. A default `cachetools` key function that hashes all args would raise `TypeError` on every call → 100% cache miss.
- Explicit `key=` lambda uses only `db_name:str` (for catalog queries) or `(platforms, infocategories, items, row_cap, db_name)` tuple (for fetch_cells). The adapter passes through the wrapper unhashed.

Purpose: Phase 2 Overview tab calls `cache.list_platforms()` on every filter change. Without cache, each change re-runs `SELECT DISTINCT PLATFORM_ID ...`. With an unprotected cache, concurrent filter changes corrupt state. This plan closes both.

Output: A single new file `app_v2/services/cache.py` (~100 lines) + test file with ≥6 tests covering hit/miss, key separation by db_name, fetch_cells key correctness, TTL expiry, and concurrent access safety.

This plan implements INFRA-08 per its ROADMAP/CONTEXT definition.
</objective>

```python
"""Thread-safe TTLCache wrappers for ufs_service _core() functions (INFRA-08).

v2.0 replacement for @st.cache_data. Every route in app_v2/ imports the wrapper
names from THIS module, never the raw _core functions from ufs_service.

Thread-safety contract (Pitfall 11):
    FastAPI `def` routes run concurrently in the threadpool. cachetools.TTLCache
    is NOT thread-safe — concurrent access during eviction can raise RuntimeError.
    Every TTLCache below is paired with a module-level threading.Lock() via the
    @cached(lock=...) parameter.

Key contract (Pitfall 11 / T-04-02):
    The DBAdapter object is NOT hashable. Including `db` in the cache key causes
    TypeError on every call (100% cache miss). All key lambdas use db_name:str
    as the cache partition identifier. Two adapters pointing at the same DB name
    share the cache (acceptable: db_name is semantically "which database" —
    identical db_name means identical data).

TTL rationale (mirrors v1.0 @st.cache_data values):
    - list_platforms: ttl=300s (catalog changes only on ingestion; ~5min is fine)
    - list_parameters: ttl=300s (same reasoning)
    - fetch_cells: ttl=60s (cell data more volatile; 1min bound for parallel
      deployment where v1.0 ingest job may write while v2.0 reads)
"""
from __future__ import annotations

import threading
from typing import Tuple

import pandas as pd
from cachetools import TTLCache, cached
from cachetools.keys import hashkey

from app.adapters.db.base import DBAdapter
from app.services.ufs_service import (
    fetch_cells_core,
    list_parameters_core,
    list_platforms_core,
)

# ---------------------------------------------------------------------------
# Module-level caches + locks. One lock per cache (cachetools supports shared
# locks across multiple caches but separate locks give finer-grained contention).
# ---------------------------------------------------------------------------

_platforms_cache: TTLCache = TTLCache(maxsize=64, ttl=300)
_platforms_lock = threading.Lock()

_parameters_cache: TTLCache = TTLCache(maxsize=64, ttl=300)
_parameters_lock = threading.Lock()

_cells_cache: TTLCache = TTLCache(maxsize=256, ttl=60)
_cells_lock = threading.Lock()


# ---------------------------------------------------------------------------
# Wrappers — same public signatures as the v1.0 @st.cache_data functions but
# with `db` (no underscore — Streamlit convention is not relevant here) and
# explicit key lambdas that exclude the adapter.
# ---------------------------------------------------------------------------

@cached(
    cache=_platforms_cache,
    lock=_platforms_lock,
    key=lambda db, db_name="": hashkey(db_name),
)
def list_platforms(db: DBAdapter, db_name: str = "") -> list[str]:
    """Return sorted distinct PLATFORM_ID values (cached per db_name).

    Key: hashkey(db_name). The adapter is NOT hashed.
    """
    return list_platforms_core(db, db_name)


@cached(
    cache=_parameters_cache,
    lock=_parameters_lock,
    key=lambda db, db_name="": hashkey(db_name),
)
def list_parameters(db: DBAdapter, db_name: str = "") -> list[dict]:
    """Return sorted distinct (InfoCategory, Item) rows (cached per db_name)."""
    return list_parameters_core(db, db_name)


@cached(
    cache=_cells_cache,
    lock=_cells_lock,
    key=lambda db, platforms, infocategories, items, row_cap=200, db_name="":
        hashkey(platforms, infocategories, items, row_cap, db_name),
)
def fetch_cells(
    db: DBAdapter,
    platforms: Tuple[str, ...],
    infocategories: Tuple[str, ...],
    items: Tuple[str, ...],
    row_cap: int = 200,
    db_name: str = "",
) -> tuple[pd.DataFrame, bool]:
    """Fetch long-form EAV rows (cached per filter tuple + db_name).

    Key: hashkey(platforms, infocategories, items, row_cap, db_name).
    The adapter is NOT hashed.

    NOTE: TTLCache returns the SAME DataFrame object to every caller until TTL
    expires. Do NOT mutate the returned DataFrame in the caller — it will be
    seen by the next caller. If mutation is needed, `df.copy()` at the call site
    (Pitfall 3 mutability concern).
    """
    return fetch_cells_core(db, platforms, infocategories, items, row_cap, db_name)


# ---------------------------------------------------------------------------
# Cache management helpers — used by tests and potentially by a future admin
# endpoint to force cache invalidation without a process restart.
# ---------------------------------------------------------------------------

def clear_all_caches() -> None:
    """Invalidate every cache. Use in tests between cases and by admin endpoints.

    Acquires each lock briefly; safe under concurrent reads (they block until
    clear completes).
    """
    for cache, lock in (
        (_platforms_cache, _platforms_lock),
        (_parameters_cache, _parameters_lock),
        (_cells_cache, _cells_lock),
    ):
        with lock:
            cache.clear()
```

def test_ttl_expiry_invalidates_entry(fake_db, monkeypatch):

## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| FastAPI threadpool → cache.py wrappers | Concurrent reads/writes from `def` routes. Every access goes through threading.Lock. |
| cache.py → ufs_service._core | Cache never sees user input directly; arguments are tuples of strings (filters) + a db_name string. Same trust model as ufs_service — user filter values pass through `sa.bindparam(..., expanding=True)` inside _core. |
| Wrapper key lambda → hashkey | The lambda explicitly includes only hashable primitives. DBAdapter object is captured but never hashed. No risk of unhashable-type TypeError under normal operation. |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-04-01 | Tampering | Concurrent TTLCache mutation during eviction | mitigate | Every `@cached` decorator passes `lock=` argument. Verified by acceptance criteria grep `threading.Lock()` count >= 3. Verified dynamically by `test_concurrent_list_platforms_no_runtime_error` test running 10 threads. |
| T-04-02 | Tampering | Cache key includes unhashable DBAdapter → 100% miss OR TypeError | mitigate | Explicit `key=lambda db, ...: hashkey(db_name, ...)` lambda on every wrapper — the `db` name is bound but not hashed. Verified by `test_list_platforms_key_excludes_adapter` test. |
| T-04-03 | Information disclosure | Returned DataFrame mutated by one caller visible to next (shared object) | accept | TTLCache deliberately returns the same object across cache hits. The contract is documented in fetch_cells docstring: "Do NOT mutate the returned DataFrame." Callers that need to mutate must `.copy()` at the call site. Same contract v1.0 honored via st.cache_data's behavior — no semantic regression. |
| T-04-04 | Information disclosure | Cross-db data bleed (two adapters with different DBs but same db_name) | accept | Cache key uses db_name, not adapter identity. If two adapters are configured with the same `name` in settings.yaml pointing at different MySQL servers, they would share cache entries. This is a misconfiguration (each DB entry in settings.yaml MUST have a unique name); treating as a configuration-layer concern. |
| T-04-05 | Denial of service | Unbounded cache growth | mitigate | Every TTLCache has maxsize (64 for catalog, 256 for fetch_cells). LRU eviction under pressure. No unbounded growth possible. |
| T-04-06 | Tampering | Cache poisoning via malicious filter values | mitigate | Filter values pass through `sa.bindparam(expanding=True)` inside `_core` — SQL injection is impossible. Cache key hashkey((platforms, ...)) accepts any tuple of strings; poisoning would require tampering at the process level (outside scope). |
| T-04-07 | Information disclosure | Leaked error str in cache exception detail | accept | Cache wrappers don't catch `_core` exceptions — they propagate to the caller (route handler), which is correct: a DB error should surface to the user via the route's error handling, not be swallowed by the cache. |
| T-04-08 | Repudiation | Stale data served after DB update | accept | TTL-based freshness (300s catalog, 60s cells). v1.0 had the same tolerance via @st.cache_data. Divergence between v1.0 and v2.0 cache clocks during parallel deployment is acceptable per ARCHITECTURE.md §"Shared State Risks". `clear_all_caches()` helper available for admin-triggered invalidation. |
