# Phase 2 — UI Design Contract: Overview Tab + Filters

> Visual and interaction contract for the first user-facing surface of PBM2 v2.0.
> Generated by gsd-ui-researcher. Verified by gsd-ui-checker.
>
> **Design system:** Bootstrap 5.3.8 (vendored). All tokens are Bootstrap defaults — no custom
> design token layer. Custom styles go in `app_v2/static/css/app.css` only when Bootstrap
> utilities cannot express the requirement.

## Spacing Scale

Bootstrap 5 uses a base-4 spacing unit (`$spacer = 1rem = 16px`).
**Do not define custom spacing tokens.** Use Bootstrap utility classes directly.

| Bootstrap Class | Computed Value | Usage in Phase 2 |
|-----------------|---------------|------------------|
| `g-2` | 8px | Filter row gutters (`row g-2`) |
| `mt-3` | 16px (1rem) | Main content top margin (inherited from `base.html`) |
| `mb-3` | 16px | Spacing below Add-platform row |
| `py-3` | 16px top+bottom | Empty state vertical breathing room |
| `py-5` | 48px top+bottom | Full-panel empty state (no platforms at all) |
| `me-auto` | auto | Spacer between left content and right buttons in entity row |
| `ms-2` | 8px | Gap between badges within a row |
| `gap-2` | 8px | Button group gap on right side of entity row |

Exceptions: none — all spacing via Bootstrap utilities.

## Component Inventory

### 1. Entity Row

**Bootstrap component:** `list-group list-group-flush` (container) + `list-group-item d-flex align-items-center` (row)

**Height:** approximately 60px per row (D-01).

**Left-to-right layout within one row (D-02):**

```
[PLATFORM_ID title fs-5 fw-bold] [Brand badge] [SoC badge] [Year badge] [me-auto spacer]
  [bi-arrow-right-circle link]  [AI Summary btn]  [× Remove btn]
```

**Element specifications:**

| Element | HTML / Bootstrap | Notes |
|---------|-----------------|-------|
| Platform ID title | `<span class="fw-bold fs-5">{{ entity.platform_id }}</span>` | Plain text, not a link |
| Navigate link | `<a href="/platforms/{{ entity.platform_id }}" aria-label="View {{ entity.platform_id }}"><i class="bi bi-arrow-right-circle"></i></a>` | Icon-only; aria-label required |
| AI Summary button | `<button class="btn btn-outline-primary btn-sm" ...><i class="bi bi-stars"></i> AI Summary</button>` | Disabled in Phase 2 (D-06) |
| Remove button | `<button class="btn btn-outline-danger btn-sm" aria-label="Remove {{ entity.platform_id }}" hx-confirm="Remove {{ entity.platform_id }} from your overview?" hx-delete="/overview/{{ entity.platform_id }}" hx-target="closest .list-group-item" hx-swap="outerHTML swap:300ms"><i class="bi bi-x"></i></button>` | Icon-only; aria-label is mandatory |

**AI Summary button state in Phase 2 (D-06):**
```html
<button class="btn btn-outline-primary btn-sm"
        disabled
        title="Content page must exist first (Phase 3)">
  <i class="bi bi-stars"></i> AI Summary
</button>
```
The `title` attribute renders as a native browser tooltip on hover. No JS tooltip library required.

**Remove HTMX contract (D-04, OVERVIEW-04):**
- `hx-delete="/overview/{{ entity.platform_id }}"`
- `hx-target="closest .list-group-item"`
- `hx-swap="outerHTML swap:300ms"` — 300ms delay allows CSS fade-out before removal
- `hx-confirm="Remove {{ entity.platform_id }} from your overview?"` — native browser confirm dialog
- On 200 response: server returns empty body; HTMX removes the element from DOM

### 2. Add Platform Row (Typeahead + Add)

**Position:** Top of `{% block content %}` in `overview/index.html`, above the filter `<details>` block (D-08).

**Layout:** Single `<form>` with `d-flex gap-2 align-items-center` containing label, full-width input, button.

```html
<form hx-post="/overview/add"
      hx-target="#overview-list"
      hx-swap="afterbegin"
      hx-on::after-request="this.reset()"
      class="d-flex gap-2 align-items-center mb-3">
  <label for="platform-input" class="form-label mb-0 text-nowrap">Add platform to overview</label>
  <input id="platform-input"
         type="text"
         name="platform_id"
         list="platforms-datalist"
         class="form-control"
         placeholder="Start typing a PLATFORM_ID…"
         autocomplete="off"
         required>
  <datalist id="platforms-datalist">
    {% for pid in all_platform_ids %}
      <option value="{{ pid }}"></option>
    {% endfor %}
  </datalist>
  <button type="submit" class="btn btn-primary text-nowrap">Add</button>
</form>
```

**`autocomplete="off"`** on the input: prevents browser autofill suggestions from overlapping the
`<datalist>` dropdown (Claude's Discretion — recommended in CONTEXT.md).

**HTMX contract (D-09):**
- `hx-post="/overview/add"` — submits form data (field name: `platform_id`)
- `hx-target="#overview-list"` — targets the list container
- `hx-swap="afterbegin"` — prepends the new row fragment at top of list
- `hx-on::after-request="this.reset()"` — clears the input after any response (success or error)

**Success response:** HTTP 200 with ONE `<li class="list-group-item ...">` fragment (single entity row).

### 3. Filter Block

**Bootstrap component:** HTML `<details>` + `<summary>` (no Bootstrap JS toggling; browser-native
open/closed state). Wrapped in a `bg-light border rounded p-2 mb-3` container.

**Open by default on first visit (D-13):** `<details open>` in the server-rendered HTML.

**localStorage persistence (D-13):** A small inline `<script>` block reads/writes `localStorage`
key `"pbm2.overview.filters.open"`. This is the **only** client-side persistence in Phase 2.

```html
<details id="filter-details" class="bg-light border rounded p-2 mb-3"
         {% if filters_open %}open{% endif %}>
  <summary class="d-flex align-items-center gap-2 user-select-none" style="cursor:pointer">
    Filters
    <span id="filter-count-badge"
          class="badge bg-primary {% if active_filter_count == 0 %}d-none{% endif %}">
      {{ active_filter_count }}
    </span>
    <a id="clear-filters-link"
       class="ms-auto small {% if active_filter_count == 0 %}d-none{% endif %}"
       href="#"
       hx-post="/overview/filter/reset"
       hx-target="#overview-list"
       hx-swap="innerHTML">Clear all</a>
  </summary>
  <form id="filter-form"
        class="row g-2 mt-2"
        hx-post="/overview/filter"
        hx-include="[data-filter]"
        hx-target="#overview-list"
        hx-swap="innerHTML">
    <div class="col-auto">
      <select name="brand" data-filter class="form-select form-select-sm" hx-trigger="change">
        <option value="">All brands</option>
        {% for brand in filter_brands %}
          <option value="{{ brand }}" {% if selected_brand == brand %}selected{% endif %}>{{ brand }}</option>
        {% endfor %}
      </select>
    </div>
    <div class="col-auto">
      <select name="soc" data-filter class="form-select form-select-sm" hx-trigger="change">
        <option value="">All SoCs</option>
        {% for soc in filter_socs %}
          <option value="{{ soc }}" {% if selected_soc == soc %}selected{% endif %}>{{ soc }}</option>
        {% endfor %}
      </select>
    </div>
    <div class="col-auto">
      <select name="year" data-filter class="form-select form-select-sm" hx-trigger="change">
        <option value="">All years</option>
        {% for year in filter_years %}
          <option value="{{ year }}" {% if selected_year == year|string %}selected{% endif %}>{{ year }}</option>
        {% endfor %}
      </select>
    </div>
    <div class="col-auto d-flex align-items-center gap-1">
      <input type="checkbox" id="has-content-check" name="has_content" value="1"
             data-filter class="form-check-input" hx-trigger="change"
             {% if selected_has_content %}checked{% endif %}>
      <label for="has-content-check" class="form-check-label small">Has content</label>
    </div>
  </form>
</details>
```

**Year dropdown sort order:** descending (newest first) — per CONTEXT.md specifics note.

**Filter `hx-trigger` placement:** Each `<select>` and the checkbox carry `hx-trigger="change"`.
The `<form>` carries `hx-post`, `hx-include`, `hx-target`, `hx-swap`. HTMX fires the request on the
element that has `hx-trigger`; `hx-include="[data-filter]"` aggregates all filter inputs into the
request body. **No debounce** for select/checkbox inputs (D-15 — these are discrete, not text).

**Active-filter count badge (D-16):**
- Server-computed (counts filters not at default value)
- Returned as an OOB swap element in the entity-list fragment response:
  ```html
  <span id="filter-count-badge" hx-swap-oob="true"
        class="badge bg-primary {% if active_filter_count == 0 %}d-none{% endif %}">
    {{ active_filter_count }}
  </span>
  ```
- `d-none` when count is 0 (badge hidden, not showing "0")

**"Clear all" link (D-17):** visible only when `active_filter_count > 0` (controlled by `d-none` class).
Click posts to `/overview/filter/reset`. Server returns the full unfiltered list + OOB badge reset to 0.

**localStorage persistence script** (placed at bottom of `overview/index.html`, outside any block):

```html
<script>
(function () {
  var el = document.getElementById('filter-details');
  if (!el) return;
  var key = 'pbm2.overview.filters.open';
  // Restore state from localStorage (overrides server-rendered default only after first visit)
  var stored = localStorage.getItem(key);
  if (stored === 'false') el.removeAttribute('open');
  else if (stored === 'true') el.setAttribute('open', '');
  // Persist on toggle
  el.addEventListener('toggle', function () {
    localStorage.setItem(key, el.open ? 'true' : 'false');
  });
}());
</script>
```

**Server-rendered default:** `<details open>` (open on first visit). After the first toggle event
the localStorage value takes precedence on subsequent loads.

### 6. Error / Alert Fragments

All error responses (409, 404) are Bootstrap alert fragments swapped into `#htmx-error-container`
via the existing `htmx-error-handler.js` from Phase 1 (D-10, D-11).

#### 6a. Duplicate platform (HTTP 409)

```html
<div class="alert alert-warning alert-dismissible fade show" role="alert">
  Already in your overview: {{ platform_id }}
  <button type="button" class="btn-close" data-bs-dismiss="alert" aria-label="Close"></button>
</div>
```

**Source:** D-10 (CONTEXT.md).

#### 6b. Unknown platform ID (HTTP 404)

```html
<div class="alert alert-danger alert-dismissible fade show" role="alert">
  Unknown platform: {{ platform_id }}. Choose from the dropdown.
  <button type="button" class="btn-close" data-bs-dismiss="alert" aria-label="Close"></button>
</div>
```

**Source:** D-11 (CONTEXT.md).

Both alerts use `alert-dismissible fade show` with a `btn-close` button so users can dismiss them.
The `htmx-error-handler.js` from Phase 1 performs the swap into `#htmx-error-container`.

| Requirement | Implementation |
