<objective>
Close gap-3 from 04-HUMAN-UAT.md (severity: minor) — Apply button does not update the trigger button count badge per D-14(b); the badge only updates on full page refresh.

The Browse page is structured so that the trigger button badges (rendered by the `picker_popover()` macro inside `_filter_bar.html`) live in the persistent `.panel-header` / `.browse-filter-bar` shell — OUTSIDE `#browse-grid`. The Apply hx-post=/browse/grid swaps INTO `#browse-grid` (innerHTML), so the persistent badge spans are never touched. POST /browse/grid currently emits only `block_names=["grid", "count_oob", "warnings_oob"]` — no fragment exists to update the trigger badges. D-14(b) is breached.

Purpose: Adopt **Candidate A (server-side OOB swap)** — extend the existing OOB pattern (`count_oob`, `warnings_oob`) with a new `picker_badges_oob` block emitting both badge spans on every POST /browse/grid response. Keep the server as the single source of truth (count comes from the same view-model that drives the grid + count caption + warnings). Mirror the proven Pattern 6 from 04-RESEARCH.md / Plan 04-03.

Why Candidate A over Candidate B (client-side JS in popover-search.js):
- Pros of A: matches the existing `count_oob` / `warnings_oob` pattern verbatim — codebase coherence, single source of truth (server count), TestClient-testable (no browser/AppTest needed). Future picker view-model changes (e.g., showing "(3 selected, 2 hidden by search)") flow through one Jinja path.
- Cons of A: ~12-15 lines of template + 1 line of router; very small.
- Pros of B: ~8 lines of JS, zero server-side change.
- Cons of B: introduces a SECOND source of truth (server count vs client checkbox-recount) for the same value — they can drift. Regression guard would have to run JS — the existing TestClient-based tests (14 of them) cannot exercise it; we would need a Playwright/AppTest setup that Phase 4 deliberately deferred (CONTEXT.md "Defer Playwright until the UI is stable (Phase 3+)"). Lower test surface == easier silent regression.

**Default: Candidate A.** This plan implements A.
