Sortable column header pattern (D-OV-07):
```jinja
{% macro sortable_th(col, label, vm) %}
  <th scope="col" role="columnheader" aria-sort="{% if vm.sort_col == col %}{{ 'descending' if vm.sort_order == 'desc' else 'ascending' }}{% else %}none{% endif %}">
    <button type="button" class="btn btn-link p-0 text-decoration-none text-reset"
            hx-post="/overview/grid"
            hx-include="#overview-filter-form"
            hx-target="#overview-grid"
            hx-swap="innerHTML swap:200ms"
            hx-vals='{"sort": "{{ col | e }}", "order": "{% if vm.sort_col == col and vm.sort_order == 'asc' %}desc{% else %}asc{% endif %}"}'
            aria-label="Sort by {{ label | e }}">
      {{ label | e }}
      {% if vm.sort_col == col %}
        {% if vm.sort_order == 'desc' %}
          <i class="bi bi-arrow-down-short"></i>
        {% else %}
          <i class="bi bi-arrow-up-short"></i>
        {% endif %}
      {% endif %}
    </button>
  </th>
{% endmacro %}
```

Sort cycle (D-OV-07): asc → desc → asc. Clicking a column that is currently the sort_col with order=asc → next order=desc; clicking the same column with order=desc → next order=asc; clicking a different column → next order=asc (default first-click on unsorted column).

The `hx-vals` JSON above implements this:
- If `vm.sort_col == col AND vm.sort_order == 'asc'` → next order is desc
- Otherwise → next order is asc
This is a 2-state cycle (asc ↔ desc) per D-OV-07's "asc → desc → asc" lock.

| AI Summary `hx-post` URL | Built from `row.platform_id`, which is regex-validated upstream (`^[A-Za-z0-9_\-]{1,128}$` — Phase 3 CONTENT-02). |
