Sort contract (D-OV-07 + D-OV-08):
- `sort_col` MUST be one of `SORTABLE_COLUMNS`; if invalid → fall back to `DEFAULT_SORT_COL`.
- `sort_order` MUST be `"asc"` or `"desc"`; if invalid → fall back to `DEFAULT_SORT_ORDER`.
- For date columns (`start`, `end`): parse ISO 8601 (`YYYY-MM-DD`). Empty/None/malformed → sentinel that sorts to END regardless of asc/desc. Implementation: use a tuple key `(is_empty: int, parsed_date: date | None, platform_id: str)` where `is_empty=1` for empty/malformed and `is_empty=0` for valid. Asc/desc applies to the `parsed_date` axis; `is_empty` always sorts non-empty first by sorting `(0, ...)` before `(1, ...)`.
- For non-date columns: case-insensitive string sort (lowercase compare); empty/None sorts to END (same `(is_empty, value, platform_id)` pattern).
- Tiebreaker: `platform_id` ASC, ALWAYS (regardless of asc/desc on the primary key).

Title fallback (D-OV-09): if `frontmatter.get("title")` is empty / missing, `OverviewRow.title = platform_id` (NOT `—`). All other missing PM fields stay `None` on the model — template (Plan 05-05) renders `None` as `—`.

`OverviewGridViewModel.filter_options[col]` extraction:
- For each filterable column, collect non-None / non-empty distinct values across ALL rows (NOT just filtered rows — filter dropdowns must show every available option, not narrow with the current filter).
- Sorted alphabetically (case-insensitive).
- The em-dash `—` sentinel is NOT a real value — it's a render-layer concern. The service-layer collection only iterates actual frontmatter values, which are either real strings or None. Filter options never contain `—`.

`OverviewGridViewModel.active_filter_counts[col]` = `len(filters.get(col, []))` for each filterable column.

    ```python
    """Overview tab orchestration — turns curated list + filters/sort into an OverviewGridViewModel.

    Single source of truth for GET /overview and POST /overview/grid (D-OV-03).
    Pure Python; NO FastAPI / Starlette imports here — routes construct
    template context dicts from the returned model.

    Contracts (from 05-CONTEXT.md):
    - D-OV-01/02: per-platform PM metadata sourced from YAML frontmatter on
      content/platforms/<pid>.md via content_store.read_frontmatter.
    - D-OV-07: 12 sortable columns; asc → desc → asc cycle; default
      start desc; tiebreaker platform_id ASC.
    - D-OV-08: dates expect ISO 8601; empty/malformed sorts to END.
    - D-OV-09: title falls back to platform_id; other missing fields → None.
    - D-OV-10: has_content_map drives AI Summary disabled state (Phase 3 D-13).
    - D-OV-12: read_frontmatter is already memoized by (pid, mtime_ns) —
      this service calls it freely per request.
    """
    from __future__ import annotations

    import datetime
    from pathlib import Path
    from typing import Literal

    from pydantic import BaseModel

    from app_v2.services.content_store import read_frontmatter
    from app_v2.services.overview_filter import has_content_file


    # 12 PM fields from D-OV-01 / D-OV-02 (column order matches table left→right).
    ALL_METADATA_KEYS: tuple[str, ...] = (
        "title", "status", "customer", "model_name", "ap_company", "ap_model",
        "device", "controller", "application", "assignee", "start", "end",
    )

    # 6 filterable columns per D-OV-13 / user lock — Title, Model Name,
    # assignee (담당자), Start, End are NOT filterable.
    FILTERABLE_COLUMNS: tuple[str, ...] = (
        "status", "customer", "ap_company", "device", "controller", "application",
    )

    # 12 sortable columns per D-OV-07.
    SORTABLE_COLUMNS: tuple[str, ...] = ALL_METADATA_KEYS

    DATE_COLUMNS: tuple[str, ...] = ("start", "end")

    DEFAULT_SORT_COL: str = "start"
    DEFAULT_SORT_ORDER: Literal["asc", "desc"] = "desc"


    class OverviewRow(BaseModel):
        """One row in the Overview pivot grid.

        Fields use `str | None = None` so missing frontmatter values render
        as None at the model layer; the template (Plan 05-05) renders None as
        the em-dash sentinel `—` (D-OV-09).

        `title` is the exception — D-OV-09 falls back to platform_id when
        frontmatter has no title field. The fallback happens in
        build_overview_grid_view_model, NOT in the model definition, so the
        model itself stays oblivious to fallback policy.
        """

        platform_id: str
        title: str
        status: str | None = None
        customer: str | None = None
        model_name: str | None = None
        ap_company: str | None = None
        ap_model: str | None = None
        device: str | None = None
        controller: str | None = None
        application: str | None = None
        assignee: str | None = None
        start: str | None = None
        end: str | None = None
        has_content: bool = False


    class OverviewGridViewModel(BaseModel):
        """View model returned by build_overview_grid_view_model."""

        rows: list[OverviewRow]
        filter_options: dict[str, list[str]]      # keys = FILTERABLE_COLUMNS
        active_filter_counts: dict[str, int]      # keys = FILTERABLE_COLUMNS
        sort_col: str
        sort_order: Literal["asc", "desc"]
        has_content_map: dict[str, bool]


    def _parse_iso_date(value: str | None) -> datetime.date | None:
        """Parse ISO 8601 'YYYY-MM-DD'. Return None on any malformed input."""
        if not value:
            return None
        try:
            return datetime.date.fromisoformat(value)
        except (ValueError, TypeError):
            return None


    def _row_sort_key(row: OverviewRow, sort_col: str, sort_order: str) -> tuple:
        """Build a sort tuple for `row` such that:
        - Empty / None / malformed values sort to END regardless of asc/desc.
        - For date columns, parsed_date is the primary key.
        - For non-date columns, lowercase string is the primary key.
        - Tiebreaker is platform_id ASC.

        The 'sort to END' invariant is enforced by `is_empty: int` as the FIRST
        tuple element: 0 for valid values, 1 for empty/malformed. Python's
        ascending tuple compare puts 0 before 1, so non-empty rows always
        come first. Reversal for desc happens AFTER sorting via list.reverse()
        on the rows array — this preserves the empty-to-END invariant
        because reverse only flips the order of the (0, ...) and (1, ...)
        groups internally; (0, ...) is still grouped together at the top.
        Wait — that's wrong. Reversing the whole list would put empties at
        the TOP for desc. We need a different approach:

        Use SEPARATE sorts: sort the non-empty subset by the primary key
        (with asc/desc applied), sort the empty subset by platform_id ASC,
        then concatenate. See _sort_rows below for the implementation.
        """
        raise NotImplementedError("This helper documents the contract; the actual sort uses _sort_rows.")


    def _sort_rows(
        rows: list[OverviewRow],
        sort_col: str,
        sort_order: Literal["asc", "desc"],
    ) -> list[OverviewRow]:
        """Sort rows so empties always go to END regardless of asc/desc.

        Algorithm:
        1. Partition rows into (non_empty, empty) by the primary sort key.
        2. Sort non_empty by primary key (asc); apply reverse() if order=desc.
        3. Sort empty by platform_id ASC (stable tail order).
        4. Return non_empty + empty.

        Both partitions use platform_id as the secondary tiebreaker.
        """
        is_date = sort_col in DATE_COLUMNS
        non_empty: list[OverviewRow] = []
        empty: list[OverviewRow] = []
        for r in rows:
            raw = getattr(r, sort_col, None)
            if is_date:
                parsed = _parse_iso_date(raw)
                if parsed is None:
                    empty.append(r)
                else:
                    non_empty.append(r)
            else:
                if raw is None or raw == "":
                    empty.append(r)
                else:
                    non_empty.append(r)

        # Sort non_empty by (primary, platform_id) asc.
        if is_date:
            non_empty.sort(key=lambda r: (_parse_iso_date(getattr(r, sort_col)), r.platform_id))
        else:
            non_empty.sort(
                key=lambda r: ((getattr(r, sort_col) or "").lower(), r.platform_id)
            )
        if sort_order == "desc":
            non_empty.reverse()
            # After reversal, the secondary tiebreaker order is also reversed
            # — fix by re-sorting same-primary-key groups by platform_id ASC.
            # Simpler approach: do a stable re-sort with platform_id ASC as
            # the FINAL sort, which preserves the relative order set by the
            # primary key (Python sort is stable).
            non_empty.sort(key=lambda r: r.platform_id)
            # Now re-apply primary key in reverse so the platform_id order is
            # preserved within each primary-key group.
            if is_date:
                # Negate by mapping date → date.toordinal() and negating int
                non_empty.sort(
                    key=lambda r: -_parse_iso_date(getattr(r, sort_col)).toordinal()
                )
            else:
                # For string desc, sort by lowercased string in reverse via
                # a stable two-pass sort: platform_id ASC, then primary key DESC.
                non_empty.sort(
                    key=lambda r: (getattr(r, sort_col) or "").lower(),
                    reverse=True,
                )

        # Empty group: always platform_id ASC.
        empty.sort(key=lambda r: r.platform_id)
        return non_empty + empty


    def _normalize_filters(filters: dict[str, list[str]] | None) -> dict[str, list[str]]:
        """Strip filters to the FILTERABLE_COLUMNS subset; convert None to {}.

        Also drops empty/whitespace-only filter values to keep the
        active_filter_counts honest.
        """
        if not filters:
            return {}
        out: dict[str, list[str]] = {}
        for col in FILTERABLE_COLUMNS:
            values = filters.get(col) or []
            clean = [v for v in values if v and v.strip()]
            if clean:
                out[col] = clean
        return out


    def _validate_sort(
        sort_col: str | None, sort_order: str | None,
    ) -> tuple[str, Literal["asc", "desc"]]:
        """Coerce sort_col / sort_order to valid values; fall back to defaults."""
        col = sort_col if sort_col in SORTABLE_COLUMNS else DEFAULT_SORT_COL
        order: Literal["asc", "desc"] = (
            sort_order if sort_order in ("asc", "desc") else DEFAULT_SORT_ORDER
        )
        return col, order


    def build_overview_grid_view_model(
        curated_pids: list[str],
        content_dir: Path,
        filters: dict[str, list[str]] | None = None,
        sort_col: str | None = None,
        sort_order: str | None = None,
    ) -> OverviewGridViewModel:
        """Pure orchestrator: read frontmatter → build rows → filter → sort.

        Args:
            curated_pids: list of PLATFORM_IDs from overview_store.load_overview()
            content_dir: filesystem path to content/platforms/
            filters: dict[col, list[selected values]] — multi-filter
            sort_col: one of SORTABLE_COLUMNS; falls back to "start"
            sort_order: "asc" or "desc"; falls back to "desc"

        Returns:
            OverviewGridViewModel with rows, filter_options, active_filter_counts,
            sort_col, sort_order, has_content_map.
        """
        col, order = _validate_sort(sort_col, sort_order)
        clean_filters = _normalize_filters(filters)

        # 1) Build all rows from frontmatter.
        all_rows: list[OverviewRow] = []
        has_content_map: dict[str, bool] = {}
        for pid in curated_pids:
            fm = read_frontmatter(pid, content_dir)
            has_content = has_content_file(pid, content_dir)
            has_content_map[pid] = has_content

            title = fm.get("title") or pid  # D-OV-09 fallback

            row = OverviewRow(
                platform_id=pid,
                title=title,
                status=fm.get("status") or None,
                customer=fm.get("customer") or None,
                model_name=fm.get("model_name") or None,
                ap_company=fm.get("ap_company") or None,
                ap_model=fm.get("ap_model") or None,
                device=fm.get("device") or None,
                controller=fm.get("controller") or None,
                application=fm.get("application") or None,
                assignee=fm.get("assignee") or None,
                start=fm.get("start") or None,
                end=fm.get("end") or None,
                has_content=has_content,
            )
            all_rows.append(row)

        # 2) Build filter_options across ALL rows (not the filtered subset).
        #    Picker dropdowns must show every available option so the user can
        #    expand their selection — narrowing on filter would create a
        #    "trapdoor" UX.
        filter_options: dict[str, list[str]] = {}
        for col_name in FILTERABLE_COLUMNS:
            values = {getattr(r, col_name) for r in all_rows}
            values.discard(None)
            values.discard("")
            # Sort case-insensitively for display.
            filter_options[col_name] = sorted(values, key=str.lower)

        # 3) Apply filters (AND across columns, OR within a column).
        def _row_matches(r: OverviewRow) -> bool:
            for fcol, selected in clean_filters.items():
                if getattr(r, fcol, None) not in selected:
                    return False
            return True

        filtered_rows = [r for r in all_rows if _row_matches(r)]

        # 4) Sort the filtered rows.
        sorted_rows = _sort_rows(filtered_rows, col, order)

        # 5) Active filter counts (always present for all 6 keys).
        active_filter_counts = {
            c: len(clean_filters.get(c, [])) for c in FILTERABLE_COLUMNS
        }

        return OverviewGridViewModel(
            rows=sorted_rows,
            filter_options=filter_options,
            active_filter_counts=active_filter_counts,
            sort_col=col,
            sort_order=order,
            has_content_map=has_content_map,
        )
    ```
