    ```python
    # ---------------------------------------------------------------------
    # Phase 5 — Overview Tab Redesign: read_frontmatter (D-OV-02, D-OV-12)
    # ---------------------------------------------------------------------
    # Defensive YAML frontmatter parser for content pages. Used by the
    # overview_grid_service (Plan 05-03) to source per-platform PM metadata.
    #
    # Contracts:
    # - NEVER raises. Every exception path returns {}.
    # - Uses yaml.safe_load EXCLUSIVELY (T-05-02-01: yaml.load is unsafe).
    # - Memoized by (platform_id, mtime_ns) — implicit invalidation when the
    #   content file changes (D-OV-12).
    # - Returns dict[str, str] — all values stringified. yaml.safe_load may
    #   yield int / float / bool / datetime.date / None for individual values;
    #   the service layer treats every metadata field as text. None values
    #   are DROPPED (key absent from result).
    # ---------------------------------------------------------------------
    import yaml as _yaml

    _FRONTMATTER_CACHE: dict[tuple[str, int], dict[str, str]] = {}


    def _parse_frontmatter_text(text: str) -> dict[str, str]:
        """Extract YAML frontmatter dict from `text`. Returns {} on any failure.

        Recognized format:
            ---\\n
            <yaml body>
            ---\\n        ← closing fence may be `---\\n` or `---` at EOF
            <body markdown — ignored>

        If `text` does not start with `---\\n`, returns {}.
        If no closing `---` is found after the leading fence, returns {}.
        """
        # Frontmatter MUST start with '---\\n' on the very first line.
        if not text.startswith("---\n"):
            return {}
        body_after_first_fence = text[4:]  # strip leading '---\\n'
        # Find the closing fence: a line that is exactly '---' (followed by
        # newline OR end-of-string).
        # Search for '\\n---\\n' first (mid-document closing fence).
        end_idx = body_after_first_fence.find("\n---\n")
        if end_idx >= 0:
            yaml_text = body_after_first_fence[:end_idx]
        else:
            # Fall back to '\\n---' at end of string (no trailing newline).
            if body_after_first_fence.endswith("\n---"):
                yaml_text = body_after_first_fence[:-4]
            else:
                return {}

        try:
            data = _yaml.safe_load(yaml_text)
        except _yaml.YAMLError:
            return {}
        except Exception:  # noqa: BLE001 — defensive; never raise on caller
            return {}

        if not isinstance(data, dict):
            # Empty fences (yaml.safe_load returns None) or non-mapping
            # top-level (list, scalar) — neither is valid frontmatter.
            return {}

        # Coerce every value to str; DROP keys whose value is None.
        result: dict[str, str] = {}
        for k, v in data.items():
            if v is None:
                continue
            result[str(k)] = str(v)
        return result


    def read_frontmatter(
        platform_id: str,
        content_dir: Path = DEFAULT_CONTENT_DIR,
    ) -> dict[str, str]:
        """Return YAML frontmatter as dict[str, str]. Returns {} on any error.

        Defensive contract (D-OV-02):
        - Missing file → {}
        - Path traversal → {}
        - No leading `---` → {}
        - Missing closing `---` → {}
        - Malformed YAML → {}
        - YAML body is not a dict (e.g. list, scalar) → {}
        - All values stringified; None values dropped

        Memoized by (platform_id, mtime_ns) per D-OV-12. The cache is
        process-local and bounded by the curated platform count (typically <
        ~100). Implicit invalidation when the file's mtime_ns changes.
        """
        try:
            mtime_ns = get_content_mtime_ns(platform_id, content_dir)
        except Exception:  # noqa: BLE001
            return {}
        if mtime_ns is None:
            # File does not exist or path traversal blocked. Do NOT cache.
            return {}

        cache_key = (platform_id, mtime_ns)
        cached = _FRONTMATTER_CACHE.get(cache_key)
        if cached is not None:
            return cached

        try:
            text = read_content(platform_id, content_dir)
        except Exception:  # noqa: BLE001
            return {}
        if text is None:
            return {}

        parsed = _parse_frontmatter_text(text)
        _FRONTMATTER_CACHE[cache_key] = parsed
        return parsed
    ```

## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| Filesystem (content/platforms/*.md) → read_frontmatter | The content files are user-authored on the platform detail page (Phase 3); the YAML frontmatter is unstructured user input. |
| read_frontmatter → callers (Plan 05-03 service, Plan 05-05 templates) | Returns dict[str, str] — no Python objects, no callables, no untrusted side effects in the parsed data. |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-05-02-01 | Tampering / RCE (YAML deserialization attack via !!python/object tags) | `yaml.load` would execute arbitrary Python on `!!python/object/apply:os.system ['rm -rf /']` style payloads | mitigate | Use `yaml.safe_load` EXCLUSIVELY. The `safe_load` parser only constructs scalars, sequences, and mappings — never Python objects. Acceptance criterion `grep -cE "\byaml\.load\b" app_v2/services/content_store.py` returns 0 enforces this. |
| T-05-02-02 | Tampering / Path traversal (PID containing `..` reads /etc/passwd) | `read_frontmatter` resolving `<content_dir>/<PID>.md` | mitigate | `read_frontmatter` calls `get_content_mtime_ns` (which calls `_safe_target` → `Path.resolve()` + `.relative_to()` — raises ValueError on traversal) BEFORE attempting to read. ValueError caught → returns `{}`. Test 7 exercises `read_frontmatter("../../etc/passwd", tmp_path)` and asserts `{}`. Defense in depth: FastAPI Path regex `^[A-Za-z0-9_\-]{1,128}$` blocks at HTTP entry too. |
| T-05-02-03 | DoS (huge YAML body — billion-laughs / yaml-bomb) | `yaml.safe_load` on a hostile YAML payload | accept | The content file size is bounded by `MAX_CONTENT_BYTES = 65536` (D-31, enforced in `save_content`). 64KB of YAML is far below the threshold for billion-laughs amplification. `safe_load` itself does not implement aliases-with-anchors expansion that the C-loader does — pure-Python `safe_load` is bounded. No mitigation beyond the existing 64KB write-time limit needed. |
| T-05-02-04 | Information disclosure (cache leaks across users) | `_FRONTMATTER_CACHE` module-level dict shared by all requests | accept | The Streamlit/FastAPI process is single-tenant on the intranet (shared-credential auth deferred). All users see the same content pages; sharing frontmatter cache is intended. No PII leak risk. If per-user lists are introduced later (auth phase), the cache key already includes `platform_id` — the cache stays correct. |
| T-05-02-05 | Tampering (cached entry served after underlying file deleted) | `_FRONTMATTER_CACHE` keyed by `(pid, mtime_ns)` | mitigate | When a file is deleted, `get_content_mtime_ns` returns `None` and `read_frontmatter` returns `{}` WITHOUT consulting the cache. The stale cache entry remains in memory but is never returned (mtime_ns lookup fails first). Memory leak is bounded by `len(curated_pids)` ≤ ~100 entries — accept. |
| T-05-02-06 | Information disclosure (frontmatter values containing HTML are rendered as raw HTML in template) | render-layer concern, not parser | mitigate (Plan 05-05) | This plan returns `dict[str, str]` — pure data. Plan 05-05's templates use Jinja2 autoescape (`| e`) on every `{{ row.title }}`, `{{ row.assignee }}`, etc. cell. T-05-05-XX in Plan 05-05's threat model addresses this directly. |

**Notes:**
- The parser is pure Python; no shell, no subprocess, no network.
- yaml.safe_load is bounded in time/memory by the 64KB content size cap.
- The cache is in-process, mtime-keyed, and bounded; no cross-request data leak.
