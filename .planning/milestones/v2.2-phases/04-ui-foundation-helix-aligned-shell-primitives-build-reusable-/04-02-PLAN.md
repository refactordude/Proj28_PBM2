    **File 3: `app_v2/static/js/chip-toggle.js`** — create new file. This is a SIBLING to `popover-search.js` (NOT a fork; popover-search.js stays byte-stable). Content:

    ```javascript
    /* Phase 4 — UI Foundation: chip-toggle helper (D-UIF-04, sibling of popover-search.js).
     *
     * Click on `.pop .opt` toggles the `.on` class on the chip and syncs
     * the value of an associated hidden <input>. Does NOT submit the form;
     * the popover's Apply button is responsible for submission.
     *
     * Loaded with `defer` AFTER bootstrap.bundle.min.js in base.html
     * (Wave 3), mirroring the popover-search.js loading pattern.
     *
     * Pitfall 8 (RESEARCH.md): popover-search.js already uses document-level
     * click delegation. Both listeners coexist via PRECISE selectors:
     *   - popover-search.js binds inside `.popover-search-root` (the
     *     existing checkbox-list popover; D-UI2-09 byte-stable).
     *   - chip-toggle.js binds on `.pop .opt` and skips clicks that fall
     *     inside `.popover-search-root` (defense-in-depth: the existing
     *     popover does NOT use .opt markup, but the early-return makes
     *     the boundary explicit and audit-friendly).
     *
     * The associated hidden <input> is located by either:
     *   1. A child of the .opt button (rare — when the input is nested
     *      inside the button for tight DOM coupling).
     *   2. A sibling within the same `.opts` group with attribute
     *      `data-opt="<chip value>"` (the convention used by
     *      filters_popover.html).
     */
    (function () {
      "use strict";

      function onChipClick(e) {
        var opt = e.target.closest('.pop .opt');
        if (!opt) return;
        // D-UI2-09 byte-stable: skip clicks inside the existing
        // checkbox-list popover-search-root. The chip-toggle popover
        // uses .pop without .popover-search-root.
        if (opt.closest('.popover-search-root')) return;

        e.preventDefault();
        opt.classList.toggle('on');

        var isOn = opt.classList.contains('on');
        // D-UIF-04 chip-value-as-payload: when ON, write the chip's data-value
        // into the hidden input (so the form submission carries the actual
        // category value); when OFF, clear the input. This diverges from
        // RESEARCH §Pitfall 8 sketch (which used '1') so that multi-option
        // groups submit a meaningful value per option, not a generic flag.
        var chipValue = opt.dataset.value || '';

        // Try nested input first; fall back to data-opt sibling lookup.
        var hidden = opt.querySelector('input[type=hidden]');
        if (!hidden && opt.parentElement) {
          hidden = opt.parentElement.querySelector(
            'input[type=hidden][data-opt="' + chipValue + '"]'
          );
        }
        if (hidden) {
          hidden.value = isOn ? chipValue : '';
        }
      }

      document.addEventListener('click', onChipClick, true);
    })();
    ```

    **Why these contents are exact:**
    - HeroSpec / FilterGroup mirror PageLink (services/joint_validation_grid_service.py:106). `Field(default_factory=list)` is required because mutable default `= []` is a Pydantic v2 anti-pattern.
    - `from __future__ import annotations` enables PEP 604 union syntax (`int | float | str`) on Python 3.13.
    - chip-toggle.js uses capture-phase (`true` as third arg) matching popover-search.js convention so the listener fires before any descendant handler can stopPropagation.
    - The data-opt sibling lookup makes filters_popover.html's hidden inputs reachable when the input is rendered as a sibling of the .opt button (the macro emits this pattern in Task 2).
    - Wave 3 will load this file via `<script src="..." defer>` in base.html.

    ```jinja
       Server-rendered inline SVG polyline. Pure Jinja — no JS, no extra
       library. Edge cases handled per RESEARCH Pitfall 4:
         - data is None or empty -> bare <svg/> (no path)
         - data has 1 element     -> degenerate horizontal line at mid-height
         - all elements equal     -> flat line at mid-height (rng = 1 fallback)

       The polyline algorithm normalizes to viewBox using min/max + 2px
       vertical padding; emits two <path> elements (filled area at 12%
       opacity + line stroke at `color`).

       Caller: `{% from "_components/sparkline.html" import sparkline %}`
               `{{ sparkline([10, 14, 18, 22, 30], width=90, height=26) }}` #}
    {% macro sparkline(data, width=90, height=26, color="var(--accent)") -%}
      {%- if not data -%}
    <svg width="{{ width }}" height="{{ height }}" aria-hidden="true"></svg>
      {%- else -%}
        {%- set lo = data | min -%}
        {%- set hi = data | max -%}
        {%- set rng = (hi - lo) if hi != lo else 1 -%}
        {%- set pad = 2 -%}
        {%- set inner_h = height - 2 * pad -%}
        {%- set step = (width / (data|length - 1)) if data|length > 1 else 0 -%}
        {%- set ns = namespace(d_line='', d_area='') -%}
        {%- for v in data -%}
          {%- set x = (loop.index0 * step) | round(2) -%}
          {%- if hi == lo -%}
            {%- set y = (height / 2) | round(2) -%}
          {%- else -%}
            {%- set y = (pad + inner_h * (1 - ((v - lo) / rng))) | round(2) -%}
          {%- endif -%}
          {%- if loop.first -%}
            {%- set ns.d_line = 'M' ~ x ~ ' ' ~ y -%}
            {%- set ns.d_area = 'M' ~ x ~ ' ' ~ y -%}
          {%- else -%}
            {%- set ns.d_line = ns.d_line ~ ' L ' ~ x ~ ' ' ~ y -%}
            {%- set ns.d_area = ns.d_area ~ ' L ' ~ x ~ ' ' ~ y -%}
          {%- endif -%}
        {%- endfor -%}
        {%- set ns.d_area = ns.d_area ~ ' L ' ~ width ~ ' ' ~ height ~ ' L 0 ' ~ height ~ ' Z' -%}
    <svg width="{{ width }}" height="{{ height }}" viewBox="0 0 {{ width }} {{ height }}" aria-hidden="true"><path d="{{ ns.d_area }}" fill="{{ color }}" opacity="0.12"/><path d="{{ ns.d_line }}" fill="none" stroke="{{ color }}" stroke-width="1.5" stroke-linecap="round" stroke-linejoin="round"/></svg>
      {%- endif -%}
    {%- endmacro %}
    ```

    Note on the `namespace(...)` pattern: Jinja's loop scope cannot reassign outer variables (loop.changed, loop.last); using a `namespace` object is the canonical idiom for accumulating state across a loop. This is documented in Jinja 3.x docs.
