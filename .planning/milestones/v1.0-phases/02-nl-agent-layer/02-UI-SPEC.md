## Typography

Inherited verbatim from Phase 1. Exactly 4 sizes, exactly 2 weights — do NOT introduce a 5th size or a 3rd weight.

| Role | Streamlit Primitive | Size | Weight | Line Height |
|------|---------------------|------|--------|-------------|
| Caption | `st.caption()` | 12px | 400 | 1.4 |
| Body / Label | `st.write()` / widget `label=` | 14px | 400 body / 600 label | 1.5 |
| Heading | `st.subheader()` | ~20px | 600 | 1.2 |
| Display | `st.title()` | ~28px | 600 | 1.1 |

Rules (unchanged from Phase 1):
- Exactly 2 weights: regular (400) and semibold (600). Do NOT use bold (700) anywhere.
- `st.title()` appears once on the Ask page: `st.title("Ask")`.
- `st.caption()` on the Ask page is reserved for: history-panel entry timestamps, history-truncation notice, and the param-confirmation count hint.
- LLM summary block is rendered with `st.write()` (14px, weight 400, line-height 1.5) — plain prose, no special size.
- Do NOT use `st.markdown("## ...")` as a substitute for `st.subheader()`.

## Color

Inherited verbatim from Phase 1. 60/30/10 split unchanged.

| Role | Value | Usage |
|------|-------|-------|
| Dominant (60%) | `#ffffff` (backgroundColor) | Main panel background, answer area |
| Secondary (30%) | `#f0f2f6` (secondaryBackgroundColor) | Sidebar background, `st.expander` backgrounds (history panel, SQL expander, partial-output expander, LLM raw output expander) |
| Accent (10%) | `#1f77b4` (primaryColor) | See reserved list below |
| Destructive | `#d62728` | SAFE-04 agent abort banner only (see copy contract) |

Accent reserved for exactly 4 elements on the Ask page:
1. The single primary CTA button: `st.button("Run Query", type="primary")` in the param-confirmation row
2. Active tab indicator in the global nav (Streamlit renders this automatically from `primaryColor`)
3. Plotly first data series — not used in Phase 2 Ask page; inherited for completeness
4. URL-copy success flash — not used on Ask page; inherited for completeness

Accent is NOT used for: the Regenerate button (secondary), history panel items, the SQL expander, warning banners, sensitivity warning, or any text.

The destructive color `#d62728` is used ONLY for the agent abort banner rendered via `st.error(...)` (SAFE-04). It is NOT used for the sensitivity warning (`st.warning`), the Ollama JSON fallback expander, or any other banner.

## Ask Page Contract

**Primary focal point:** NL question input (`st.text_area`) at the top of the main panel.

**Page entry point file:** `app/pages/ask.py`

**Session state keys owned by this page** (namespace `ask.*`):
- `ask.question` — current question text
- `ask.history` — list of history entries (max 50)
- `ask.openai_warning_dismissed` — bool, session-scoped
- `ask.confirmed_params` — list of confirmed `(InfoCategory, Item)` tuples after NL-05
- `ask.last_sql` — str, last executed SQL
- `ask.last_df` — DataFrame, last result
- `ask.last_summary` — str, last LLM summary
- `ask.pending_params` — list proposed by agent, pre-confirmation

### Top-of-Page Banner Zone

Banners render at the very top of the main panel, above all other content. At most one banner is shown at a time. Priority order (highest first):

1. SAFE-04 abort banner (`st.error`) — shown only when agent aborts mid-session
2. NL-10 sensitivity warning (`st.warning`) — shown when OpenAI is active and not yet dismissed this session

Both banners are defined in the copy contract below.

### Full Page Layout (top to bottom, from D-18, D-19)

```
[Banner zone — st.error or st.warning, conditional]

st.title("Ask")

st.expander("History ({N})", expanded=False)   ← D-19: collapsed by default, above input
  └─ [history entries — see History Panel section]

st.text_area("Your question", ...)              ← primary focal point

[Param confirmation row — st.multiselect + st.button("Run Query"), conditional on NL-05]

[Answer zone — rendered only after a successful run]
  ├─ st.dataframe(result_df, ...)
  ├─ st.write(llm_summary)
  └─ st.expander("Generated SQL", expanded=False)
       ├─ st.code(sql, language="sql")
       └─ st.button("Regenerate", type="secondary")   ← D-20

[Starter prompt gallery — rendered only when no question has been submitted yet]
  └─ 8 st.button prompts in a 4-column grid
```

## NL Question Input

```python
question = st.text_area(
    "Your question",
    value=st.session_state.get("ask.question", ""),
    placeholder="e.g. What is the WriteProt status for all LUNs on platform X?",
    height=80,
    key="ask.question",
)
```

**Loading state** (while agent is running):
```python
with st.spinner("Thinking..."):
    ...  # agent call
```
Copy is exactly `"Thinking..."` — not "Running query…" or "Fetching…".

**Empty state** (no question submitted yet): show starter prompt gallery (see Starter Prompt Gallery section). Do not render an `st.info` message in addition to the gallery.

**Error state** (agent raised an unhandled exception, distinct from SAFE-04 abort):
```python
st.error("Something went wrong. Try rephrasing your question. ({short_error_type})")
```
Copy template: `"Something went wrong. Try rephrasing your question. ({ExceptionType})"` where `{ExceptionType}` is the Python exception class name only (e.g. `TimeoutError`), not the full message (which may contain path data).

# Candidate params: list of "InfoCategory / Item" label strings, pre-populated from agent proposal
confirmed = st.multiselect(

## Answer Zone

Rendered only after a successful agent run (result table + summary available).

### Result Table

```python
st.dataframe(
    result_df,
    use_container_width=True,
    hide_index=True,
    column_config={col: st.column_config.TextColumn(col) for col in result_df.columns},
)
```

Row-count indicator below the table:
```python
st.caption(f"{len(result_df)} rows returned.")
```

Row-cap warning (when `AgentConfig.row_cap` reached):
```python
st.warning("Result capped at {row_cap} rows. Refine your question to see all data.")
```
Placed above the result table, not below it. Copy template uses the actual `row_cap` value from `AgentConfig`.

### LLM Summary Block

```python
st.write(llm_summary)   # plain-text prose, weight 400, 14px, line-height 1.5
```

No heading above the summary — the position in the narrative flow (below the table, above the SQL expander) provides sufficient context. Do NOT wrap in `st.info` or `st.expander`.

### SQL Expander (NL-02, D-20)

```python
with st.expander("Generated SQL", expanded=False):
    st.code(sql, language="sql")
    st.button("Regenerate", type="secondary", key="ask.regenerate")
```

- Expander is collapsed by default on every page render.
- "Regenerate" is a `type="secondary"` button (not accent-colored).
- Clicking "Regenerate" reuses `st.session_state["ask.question"]` + `st.session_state["ask.confirmed_params"]`, issues a fresh LLM call, and replaces the current answer in place.
- No confirmation dialog before Regenerate — it is non-destructive (old result is replaced with a new one).

### Ollama JSON Fallback Expander (D-23)

Rendered only when the Ollama fallback chain was exercised (power-user debugging):

```python
with st.expander("LLM raw output", expanded=False):
    st.code(raw_llm_text, language=None)
```

No user-facing warning is shown unless ALL fallbacks fail. If all fallbacks fail, render the question-level error state (see NL Question Input error copy above).

## History Panel (NL-04, D-19)

```python
history_count = len(st.session_state.get("ask.history", []))
with st.expander(f"History ({history_count})", expanded=False):
    if history_count == 0:
        st.caption("Your recent questions appear here.")
    else:
        for entry in reversed(st.session_state["ask.history"]):
            col_q, col_ts = st.columns([5, 1])
            with col_q:
                label = entry["question"][:80] + "..." if len(entry["question"]) > 80 else entry["question"]
                status_icon = "" if entry["status"] == "ok" else " ✗"
                if st.button(label + status_icon, key=f"history_{entry['id']}", type="secondary"):
                    st.session_state["ask.question"] = entry["question"]
                    st.rerun()
            with col_ts:
                st.caption(entry["timestamp"])   # "HH:MM" format, session-relative
```

- History is session-only; cleared on browser refresh (D-19).
- Entries with `status: "failed"` show a ✗ suffix on the button label (plain Unicode, not `st.error`).
- Clicking a history entry refills the question text area and triggers a rerun (D-19 specifies re-runs immediately — `st.rerun()` after writing to session state).
- Cap at 50 entries. When the 51st entry is added, drop the oldest (LRU). <!-- ASSUMED: LRU chosen over hard-stop because it is less surprising; user never sees a "full" error -->
- Truncation notice: `st.caption("History truncated to 50 most recent questions.")` rendered at the bottom of the expander content when the cap has been hit at least once this session.

**History entry empty state copy:** `"Your recent questions appear here."`

## Agent Abort Banner (SAFE-04, D-22)

Rendered via `st.error()` at the top of the main panel (banner zone, priority 1) when the agent exceeds `max_steps=5` or `timeout_s=30`.

Two distinct copy variants — use the one that matches the abort reason:

**Step-cap abort:**
```python
st.error("Agent stopped: reached the 5-step limit. Try rephrasing your question with more specific parameters.")
```

**Timeout abort:**
```python
st.error("Agent stopped: query timed out after 30 seconds. Try a more targeted question or switch to a faster model.")
```

Below the banner, render the partial output expander (D-22):
```python
with st.expander("Partial output", expanded=False):
    if last_tool_call:
        st.subheader("Last tool call")
        st.code(last_tool_call, language="python")
    if partial_df is not None and not partial_df.empty:
        st.subheader("Retrieved rows")
        st.dataframe(partial_df, use_container_width=True, hide_index=True)
```

The failed history entry is written with `status: "failed"` so Regenerate remains available. The Regenerate button renders in the SQL expander as normal — the abort banner does not suppress it.

Destructive color `#d62728` applies here via Streamlit's `st.error()` primitive. No other banner on this page uses this color.

## OpenAI Sensitivity Warning (NL-10, D-25)

Rendered via `st.warning()` at the top of the main panel (banner zone, priority 2) when:
- `st.session_state["active_llm"] == "openai"`, AND
- `st.session_state.get("ask.openai_warning_dismissed")` is falsy

**Exact copy (from D-25):**
```python
st.warning(
    "You're about to send UFS parameter data to OpenAI's servers. "
    "Switch to Ollama in the sidebar for local processing."
)
```

Dismissible: render a `st.button("Dismiss", type="secondary", key="ask.dismiss_warning")` directly below the `st.warning`. On click: `st.session_state["ask.openai_warning_dismissed"] = True; st.rerun()`.

- Re-shown after browser refresh (session state is cleared).
- NOT shown when `active_llm == "ollama"`.
- NOT shown when `ask.openai_warning_dismissed` is `True`.
- The Dismiss button uses the exact label `"Dismiss"` (verb + implicit object — short and clear).

## Starter Prompt Gallery (ONBD-01, ONBD-02, D-27)

Shown only on initial page load (before the user has submitted any question in this session):

```python
if not st.session_state.get("ask.history"):
    st.subheader("Try asking...")
    prompts = load_starter_prompts()  # reads config/starter_prompts.yaml
    cols = st.columns(4)
    for i, prompt in enumerate(prompts[:8]):
        with cols[i % 4]:
            if st.button(prompt["label"], key=f"starter_{i}", type="secondary"):
                st.session_state["ask.question"] = prompt["question"]
                st.rerun()
```

Layout: 8 prompts in a 4-column grid (2 rows × 4 columns). <!-- ASSUMED: 4 columns fits comfortably on a 1280px+ intranet desktop display; 2 rows keeps the gallery compact above the question input -->

Each button uses the `type="secondary"` style (not accent-colored — these are suggestions, not the primary CTA).

Prompt `label` is a short title (≤ 40 chars); `prompt["question"]` is the full question text pre-filled into the text area. The user still presses "Run Query" (or submits the form) — no auto-submit on gallery click (D-27).

Heading: `st.subheader("Try asking...")` — weight 600, ~20px.

The gallery disappears once `ask.history` has any entry (i.e., after the first successful or failed run). This is intentional — the gallery gives way to the history panel for repeat users.

**Starter prompt YAML structure** (for `config/starter_prompts.example.yaml`):
```yaml
- label: "WriteProt status by platform"
  question: "What is the WriteProt status for all LUNs on each platform?"
- label: "Compare bkops across platforms"
  question: "Compare background operations settings across all platforms."
# ... 6 more entries
```

## Interaction States Contract

Every data-loading surface on the Ask page must implement all four states, consistent with Phase 1.

| State | Primitive | Trigger |
|-------|-----------|---------|
| Loading | `st.spinner("Thinking...")` | While agent call is in-flight (question submitted through to result) |
| Empty | Gallery (pre-first-run) or `st.caption("Your recent questions appear here.")` (history) | No question yet submitted; empty history |
| Error | `st.error(...)` | Agent abort (SAFE-04), unhandled exception; see copy above |
| Success | `st.dataframe(...)` + `st.write(summary)` + SQL expander | Agent returned a `SQLResult` |

The sensitivity warning is a persistent informational banner, not one of the four interaction states. It renders regardless of loading/success/error/empty status when conditions are met.

## Accessibility Contract

Unchanged from Phase 1. Applied to Phase 2 as follows:

- `st.text_area` call MUST include `label="Your question"` (non-empty, visible by default).
- `st.multiselect` for param confirmation MUST include `label="Parameters to include"`.
- History entry buttons must include meaningful label text (the question text, truncated to 80 chars + `...`).
- Color is never the sole indicator of state: failed history entries use ✗ text AND the label suffix (not color alone). Abort banner uses both `st.error` color AND explicit copy.
- All `st.subheader()` calls on this page: `"Try asking..."` only (one heading per page rule).
