# Phase 4 — UI Design Contract: Browse Tab Port

> Visual and interaction contract for the pivot-grid Browse tab: filter bar, popover-checklist multiselect,
> sticky-header table, cap warnings, empty state, and URL round-trip.
>
> Generated by gsd-ui-researcher. Verified by gsd-ui-checker.
>
> **Design system:** Bootstrap 5.3.8 (vendored) + Dashboard token layer established in Phase 03
> (`app_v2/static/css/tokens.css` + `app_v2/static/css/app.css`). Phase 04 extends `app.css` with
> `.pivot-table` rules and a scroll-container override on `.panel-body`. All other visual treatment
> inherits from Phase 03 unchanged.
>
> **BROWSE-V2-04 scope note:** Excel/CSV export has been removed from Phase 4 scope per D-19..D-22
> (user decision 2026-04-26). No export button, no export endpoint, no openpyxl under `app_v2/`.

**Pivot cell typography rule:**
- Font: `"JetBrains Mono", ui-monospace, monospace` (D-27)
- Size: 13px (denser than 15px body for wide-form readability at 30 columns)
- `white-space: nowrap` — prevents cell wrapping; horizontal scroll on the table-responsive container handles overflow
- No color override — cells render `var(--ink)` (#171c24); blank/null cells render empty string (not "null" or "—")

**Column header typography rule:**
- Font: Inter Tight (inherits from body)
- Size: 12px
- Weight: 600
- Color: `var(--mute)` (#8b93a0) — same as Dashboard `.jvt thead th` pattern
- Text-transform: none (PLATFORM_IDs are mixed-case identifiers; parameter labels use sentence-case)
- `white-space: nowrap`

## Color

Phase 04 inherits the full Phase 03 color contract. No new color tokens. The 60/30/10 rule continues to
apply; the Browse page uses all three surface roles and adds one semantic alert color from the existing
token palette.

| Role | Token / Bootstrap | Computed | Usage in Phase 4 |
|------|------------------|----------|------------------|
| Dominant (60%) — page surface | `var(--bg)` | `#f3f4f6` | Body background (outside panel) |
| Secondary (30%) — panel surfaces | `var(--panel)` | `#ffffff` | Browse `.panel` background; popover `.dropdown-menu` background |
| Accent (10%) — primary actions | `var(--accent)` ↔ `btn-primary` | `#3366ff` | `[Apply]` button inside each picker popover footer |
| Warning (recoverable) | `var(--amber)` ↔ `alert-warning` | `#f59e0b` | Row-cap warning; col-cap warning (D-24) |
| Informational | Bootstrap `alert-info` | Bootstrap default | Empty-state alert (D-25) — no custom override |
| Secondary surface (popover footer) | Bootstrap `.bg-light` | `#f8f9fa` | Sticky popover footer background |
| Muted / helper | `var(--mute)` ↔ `text-muted` | `#8b93a0` | Count caption, column headers, badge muted text |
| Ink | `var(--ink)` | `#171c24` | Pivot cell text, panel heading, checklist option labels |
| Selection count badge | `bg-secondary` | Bootstrap `#6c757d` | Count badge on trigger buttons when selection is active |
| Border | `var(--line)` | `#eaecef` | Hairlines inside popover, panel header bottom border |
| Border (stronger) | `var(--line-2)` | `#e1e4e8` | Thead bottom border (`border-bottom: 2px solid var(--line-2)`) |
| Thead sticky background | `var(--bs-light)` (Bootstrap) | `#f8f9fa` | `<thead class="sticky-top bg-light">` (D-26) |

**Accent (#3366ff) reserved exclusively for (Phase 04 additions in bold):**
1. **`[Apply (N)]` button inside picker popover footer** (`btn btn-primary btn-sm`)
2. Active nav-link (inherited from `base.html` Phase 1)
3. Brand badge (inherited from Phase 02)
4. Save button on edit view (inherited from Phase 03)
5. Add Content button on empty state (inherited from Phase 03)
6. Active nav-pill (inherited from Phase 03)
7. Inline links in rendered markdown (inherited from Phase 03)

**Warning (#f59e0b) reserved exclusively for:**
1. Row-cap alert: "Result capped at 200 rows…" (D-24)
2. Col-cap alert: "Showing first 30 of {N} parameters…" (D-24)
3. LLM failure alert in summary slot (inherited from Phase 03)

**Why informational (Bootstrap alert-info, not accent) for empty state:**
The empty state (D-25) is a neutral instruction — the user has not done anything wrong and there is nothing
to recover from. Blue-tone `alert-info` signals "waiting for input," which matches the intent exactly.
Using `var(--accent)` here would imply a CTA or primary action, which the empty state is not.

**No violet (#7a5af8) on Browse page:** Browse has no AI affordances. Violet remains exclusively for
`.ai-btn` / Regenerate button (AI-only reservation from Phase 03).

### 4. Picker Popover (Jinja Macro)

Reusable macro, called identically for Platforms and Parameters. Full list rendered server-side on page
load (D-11). Client-side substring filter via `popover-search.js` (D-10).

```jinja
{# app_v2/templates/browse/_picker_popover.html #}
{% macro picker_popover(name, label, options, selected) %}
<div class="dropdown">
  <button class="btn btn-outline-secondary btn-sm dropdown-toggle"
          type="button"
          id="picker-{{ name }}-trigger"
          data-bs-toggle="dropdown"
          data-bs-auto-close="outside"
          aria-expanded="false"
          aria-label="Select {{ label | lower }}{% if selected %} ({{ selected | length }} selected){% endif %}">
    {{ label }}
    {% if selected %}
      <span class="badge bg-secondary ms-1">{{ selected | length }}</span>
    {% endif %}
    <i class="bi bi-chevron-down ms-1"></i>
  </button>

  <div class="dropdown-menu p-0 popover-search-root"
       style="min-width: 320px; max-width: 480px;"
       aria-labelledby="picker-{{ name }}-trigger">

    {# Header: search input (D-09) #}
    <div class="p-2 border-bottom">
      <input type="search"
             class="form-control form-control-sm popover-search-input"
             placeholder="Search {{ label | lower }}…"
             autocomplete="off"
             aria-label="Search {{ label | lower }}">
    </div>

    {# Body: scrollable checklist (D-09) #}
    <ul class="list-unstyled m-0 popover-search-list"
        style="max-height: 320px; overflow-y: auto;"
        role="listbox"
        aria-label="{{ label }} options">
      {% for opt in options %}
        <li data-label="{{ opt | e }}" role="option">
          <label class="dropdown-item d-flex align-items-center gap-2"
                 title="{{ opt | e }}">
            <input type="checkbox"
                   class="form-check-input flex-none"
                   name="{{ name }}"
                   value="{{ opt | e }}"
                   form="browse-filter-form"
                   {% if opt in selected %}checked{% endif %}>
            <span class="text-truncate">{{ opt | e }}</span>
          </label>
        </li>
      {% endfor %}
    </ul>

    <div class="p-2 border-top d-flex justify-content-between align-items-center bg-light sticky-bottom">
      <button type="button"
              class="btn btn-link btn-sm popover-clear-btn text-secondary"
              aria-label="Clear {{ label | lower }} selection">
        Clear
      </button>
      <button type="button"
              class="btn btn-primary btn-sm popover-apply-btn"
              hx-post="/browse/grid"
              hx-include="#browse-filter-form input:checked"
              hx-target="#browse-grid"
              hx-swap="innerHTML swap:200ms"
              hx-on:click="bootstrap.Dropdown.getInstance(document.getElementById('picker-{{ name }}-trigger')).hide()"

```

**Visual spec — popover dimensions:**
- Width: `min-width: 320px; max-width: 480px` (Claude's Discretion resolution from CONTEXT.md)
- Border-radius: Bootstrap default for `.dropdown-menu` (6px)
- Shadow: Bootstrap default (`box-shadow: 0 0.5rem 1rem rgba(0,0,0,.15)`)

**Label truncation:**
- `text-truncate` (Bootstrap) clips long labels with ellipsis at the popover width
- `title="{{ opt | e }}"` on the `<li>` provides the full label on hover (browser tooltip)

**Apply button count:**
- Shows the current checkbox count inside the button: `Apply 3` (with badge)
- Count badge is updated by `popover-search.js` `onCheckboxChange` handler
- `badge bg-light text-primary` — light-fill badge using accent text, consistent with Dashboard "active count" treatment

**Source:** D-07, D-08, D-09, D-10, D-11, D-14, D-15 (CONTEXT.md); Pattern 3, Pattern 4 (RESEARCH.md);
Claude's Discretion (CONTEXT.md): 320/480px width, label truncation, count on Apply, autocomplete="off".

### 5. `popover-search.js` — Client-side Substring Filter

~55 lines vanilla JS. Event-delegated on `document`. No dependencies. Attached to all
`.popover-search-root` widgets.

**Responsibilities:**
1. `onInput` — hides `<li>` rows where `data-label.toLowerCase()` does not include the search query
2. `onCheckboxChange` — updates the `.popover-apply-count` badge as checkboxes toggle
3. `onClearClick` — empties all checkboxes within the popover root; fires synthetic change events to update count; does NOT trigger HTMX (D-15)
4. `onDropdownShow` — stashes current checked state to `root.dataset.originalSelection` (D-15 restore-on-close); focuses search input on open (D-09)
5. `onDropdownHide` — restores original selection if Apply was not clicked (D-15)
6. `onApplyClick` — marks `root.dataset.applied = "1"` so `onDropdownHide` knows not to restore

**Wire-up in `base.html`:**

```html
<script src="{{ url_for('static', path='js/popover-search.js') }}" defer></script>
```

**Source:** D-10, D-14, D-15 (CONTEXT.md); Pattern 4 (RESEARCH.md).

### 6. Pivot Table

The `#browse-grid` target holds warnings (if any) followed by the table or empty state. The table is inside

```html
{# app_v2/templates/browse/_grid.html #}
<div class="table-responsive">
  <table class="table table-striped table-hover table-sm pivot-table">
    <thead class="sticky-top bg-light">
      <tr>
        <th scope="col">{{ vm.index_col_name | e }}</th>
        {% for col in vm.df_wide.columns if col != vm.index_col_name %}
          <th scope="col">{{ col | e }}</th>

    </thead>
    <tbody>
      {% for _, row in vm.df_wide.iterrows() %}
        <tr>
          {% for col in vm.df_wide.columns %}
            <td>{{ row[col] | string | e if row[col] is not none else "" }}</td>
          {% endfor %}
        </tr>
      {% endfor %}

```

**Cell rendering rules (D-27):**
- Every cell rendered as plain text: `row[col] | string | e`
- Null / None values render as empty string `""` — the `.pivot-table td:empty::after` CSS injects an em-dash
- No column-type configuration — EAV `Result` values are heterogeneous (hex, decimal, error strings, CSV)
- Font: `"JetBrains Mono"` (applied by `.pivot-table td` rule)

**Sticky header (D-26, Pattern 5):**
- `<thead class="sticky-top bg-light">` — Bootstrap `sticky-top` applies `position: sticky; top: 0`
- Sticks to `.browse-grid-body` (the `overflow-y: auto` scroll container), not the viewport
- `z-index: 2` on `.pivot-table thead.sticky-top` — above tbody (z-index: auto), below Bootstrap
  dropdowns (z-index: 1000) and the sticky popover footer

**Horizontal scroll (D-28):**
- `<div class="table-responsive">` provides `overflow-x: auto`
- No sticky-left first column in this phase (deferred per D-28 decision)

**Source:** D-26, D-27, D-28, D-29 (CONTEXT.md); Pattern 5 (RESEARCH.md); BROWSE-V2-02 (REQUIREMENTS.md).

## HTMX Behavior Summary

| Element | Trigger | Method + Endpoint | Target | Swap |
|---------|---------|-------------------|--------|------|
| Apply button (Platforms picker) | click | `POST /browse/grid` | `#browse-grid` | `innerHTML swap:200ms` |
| Apply button (Parameters picker) | click | `POST /browse/grid` | `#browse-grid` | `innerHTML swap:200ms` |
| Swap-axes toggle (`btn-check`) | change | `POST /browse/grid` | `#browse-grid` | `innerHTML swap:200ms` |
| Clear-all link | click | `POST /browse/grid` (empty body) | `#browse-grid` | `innerHTML swap:200ms` |
| Grid responses (all above) | — | — | `#grid-count` (OOB) | `hx-swap-oob="true"` |
| Errors (4xx/5xx from any route) | automatic (`htmx-error-handler.js`) | — | `#htmx-error-container` | `innerHTML` |

**`hx-include` on Apply buttons:**
- `#browse-filter-form input:checked` — aggregates ALL checked checkboxes from both pickers (they
  associate to the form via `form="browse-filter-form"` attribute, even though they're DOM-children of the
  `.dropdown-menu` elements)

**`hx-include` on Swap toggle:**
- `#browse-filter-form input[name='platforms']:checked, #browse-filter-form input[name='params']:checked, #browse-swap-axes:checked`
- Uses name-specific selector to pick only platform and param checkboxes, plus the swap checkbox itself

**`hx-push-url="true"`:** NOT set directly on any element. Instead, the POST `/browse/grid` route returns
`HX-Push-Url: /browse?platforms=...&params=...&swap=1` response header (D-32). This sets the address bar to
the canonical `/browse` URL, not `/browse/grid`.

**Swap animation:** `hx-swap="innerHTML swap:200ms"` on all grid-swapping elements produces a subtle 200ms
fade (Claude's Discretion from CONTEXT.md accepted).

**No `hx-push-url` on popover internal Clear button** — Clear does not fire HTMX at all (D-15).

**Source:** D-14, D-16, D-17, D-18, D-32 (CONTEXT.md); Pattern 3 (RESEARCH.md).

## URL Round-Trip Contract (BROWSE-V2-05)

| Parameter | Format | Example | FastAPI Type |
|-----------|--------|---------|-------------|
| `platforms` | Repeated key | `?platforms=Samsung_S22U_SM8450&platforms=Pixel_7_Tensor` | `Annotated[list[str], Query(default_factory=list)]` |
| `params` | Repeated key | `?params=attribute+%C2%B7+vendor_id` | Same |
| `swap` | `"1"` or omitted | `?swap=1` | `Annotated[str, Query()] = ""` |

**Encoding:** FastAPI/Starlette URL-encodes on the wire. Middle-dot separator ` · ` (U+00B7) encodes to
`%20%C2%B7%20`. Server splits on the literal ` · ` after decoding (D-33).

**Initial GET reads query params:** `GET /browse?platforms=A&params=B&swap=1` pre-checks popover
checkboxes AND renders the filtered grid — same `build_view_model(...)` call that POST uses.

**URL updated on every filter change:** POST `/browse/grid` sets `HX-Push-Url` response header to the
canonical `/browse?...` URL (D-32). User can copy the URL from the address bar and share it.

**Source:** D-30, D-31, D-32, D-33 (CONTEXT.md); Pattern 2 (RESEARCH.md); BROWSE-V2-05 (REQUIREMENTS.md).

## State Diagram — Browse Tab

```
[Empty state]
  "Select platforms and parameters above…"
       |
   [Select platforms AND parameters → Apply]
       |
       ↓
[Pivot grid rendered]
  "N platforms × K parameters"
       |
   ┌───┴────────────────────────────────────────┐
   │                                            │
[Hit row cap (200)]                   [Hit col cap (30)]
   ↓                                            ↓
[Grid + row-cap warning]            [Grid + col-cap warning]
"Result capped at 200 rows…"        "Showing first 30 of N…"
   │                                            │
   └───────────────────┬────────────────────────┘
                       │
              Both caps possible simultaneously
                       │
               [Clear all]
                       ↓
               [Empty state]
```

**State transitions:**
- Every filter Apply → POST `/browse/grid` → `innerHTML` swap on `#browse-grid`
- Every Swap toggle → POST `/browse/grid` → same swap
- Clear all → POST `/browse/grid` (empty body) → Empty state
- Any of the above also OOB-swaps `#grid-count`
- All transitions update the URL bar via `HX-Push-Url` response header

**No error state on Browse tab itself** — if the DB query fails, FastAPI raises a 500 which HTMX routes
to `#htmx-error-container` via `htmx-error-handler.js` (Phase 1 global handler). The grid slot keeps its
last state; the error container shows the message.

## Accessibility Checklist

| Requirement | Implementation |
|-------------|---------------|
| Picker trigger buttons labelled | `aria-label` includes selection count: "Select platforms (3 selected)" |
| Dropdown expanded state | Bootstrap applies `aria-expanded` automatically on `.dropdown-toggle` |
| Checklist role | `ul.popover-search-list` carries `role="listbox"`; each `<li>` carries `role="option"` |
| Checkboxes labelled | Each `<input type="checkbox">` is the first child of a `<label>` element (implicit association) |
| Alert roles | `.alert-warning` and `.alert-info` elements carry `role="alert"` — announced by screen readers |
| Swap toggle labelled | `aria-label="Swap axes (rows and columns)"` on `<input type="checkbox" class="btn-check">` |
| Clear-all labelled | `aria-label="Clear all filters"` on the `<a>` element |
| Apply / Clear buttons labelled | `aria-label` specifies which picker they act on |
| Search input labelled | `aria-label="Search platforms…"` / `aria-label="Search parameters…"` on the `<input type="search">` |
| Table column headers | `<th scope="col">` on all header cells |
| Table body index column | First `<td>` of each row is a cell (not a header); no `scope` needed since headers cover columns not rows |
| Count caption live region | `#grid-count` is updated via HTMX OOB swap; add `aria-live="polite"` so screen readers announce the updated count |
| Color not sole differentiator | Cap warnings have `bi-exclamation-triangle` icon + "Result capped…" prefix text; empty state has descriptive copy — no color-only signaling |
| Keyboard popover navigation | Bootstrap dropdown provides keyboard navigation (Enter to open, Esc to close, arrow keys within menu) |
| Focus management after swap | After grid swap, HTMX does not move focus (correct — the filter bar stays in view and focus remains on the Apply button that was just clicked) |
| Pivot table scrolling | `<div class="table-responsive">` + `.browse-grid-body` are scrollable via keyboard (they become scroll containers; `tabindex="0"` should be added to `.browse-grid-body` to make keyboard scrolling discoverable) |
| Font size floor | 13px minimum in pivot cells — above the 12px WCAG floor; 15px for chrome text |

**`tabindex="0"` on `.browse-grid-body`:**

```html
<div class="panel-body browse-grid-body" tabindex="0" aria-label="Pivot grid results">
```

This makes the scroll container keyboard-focusable (Tab reaches it) so keyboard users can scroll the table
without a mouse.
