### Font Loading Fix (load-bearing for this phase)

`base.html` currently has NO Google Fonts `<link>` tag. `tokens.css` references `"Inter Tight"` in the font-family stack but relies on system fallback. `app.css` already uses `font-weight: 800` for `.page-title`. Phase 4 MUST add the following to `<head>` in `base.html` before the tokens.css link:

```html
<link rel="preconnect" href="https://fonts.googleapis.com"/>
<link rel="preconnect" href="https://fonts.gstatic.com" crossorigin/>
<link href="https://fonts.googleapis.com/css2?family=Inter+Tight:wght@400;500;600;700;800&family=JetBrains+Mono:wght@400;500;600&display=swap" rel="stylesheet"/>
```

Source: Dashboard_v2.html lines 7–9. Weight range 400–800 is required: `brand-mark` and `page-title` use 800 (Dashboard lines 36, 65); tabs and `.ph b` use 700 (lines 51, 119); body uses 400 (line 25).

## New Jinja Macros

All macros live in `app_v2/templates/_components/`. Each file exports a single `{% macro name(...) %}`.

### `_components/topbar.html` — `topbar(active_tab)`

Renders the Helix topbar. Tabs are hardcoded to the PBM2 trio: Joint Validation (`href="/"`, tab id `overview`), Browse (`href="/browse"`, tab id `browse`), Ask (`href="/ask"`, tab id `ask`). Brand-mark renders the letter "P" (not "H"). Avatar slot shows static "PM" initials.

Macro signature:
```jinja
{% macro topbar(active_tab="") %}
```

```jinja
{% macro page_head(title, subtitle="", actions_html="") %}
```

### `_components/hero.html` — `hero(spec)`

Renders the hero card. Takes a `HeroSpec` Pydantic model instance. Gracefully renders with empty `segments` (no segmented bar row) or empty `side_stats` (no side panel — collapses to single-column layout). Macro reads `spec.label`, `spec.big_number`, `spec.big_number_unit`, `spec.delta_text`, `spec.segments`, `spec.side_stats`.

Macro signature:
```jinja
{% macro hero(spec) %}
```

### `_components/kpi_card.html` — `kpi_card(label, value, unit="", delta="", delta_tone="flat", spark_data=None, variant="4-up")`

ONE macro handles both grid variants. The `variant` arg (`"4-up"` or `"5-up"`) is used by the CALLER to set the container class (`.kpis` or `.kpis.five`) — the macro itself renders a single `.kpi` card and doesn't set the grid. `spark_data` is a list of numbers passed to the sparkline sub-macro. `delta_tone` values: `"up"`, `"down"`, `"flat"`, `"ok"`.

Macro signature:
```jinja
{% macro kpi_card(label, value, unit="", delta="", delta_tone="flat", spark_data=None) %}
```

### `_components/sparkline.html` — `sparkline(data, width=90, height=26, color="var(--accent)")`

Server-rendered inline SVG polyline. Algorithm: normalize data to viewBox using min/max + 4px padding; emit two `<path>` elements (filled area at 12% opacity + line stroke at `color`). Edge cases:
- `data` is empty or None → `<svg width="{width}" height="{height}"/>` (no paths)
- `data` has exactly 1 element → degenerate horizontal line at mid-height
- All elements are equal (constant) → flat line at mid-height

Color defaults to `var(--accent)`. Planner passes the CSS variable so future restyles change the token, not the template.

Macro signature:
```jinja
{% macro sparkline(data, width=90, height=26, color="var(--accent)") %}
```

### `_components/date_range_popover.html` — `date_range_popover(form_id, field_prefix="date", quick_days=[7,14,30,60], start_val="", end_val="")`

Bootstrap 5 dropdown wrapping `.pop` content. Quick-chip row with the given day options (`.qrow button.active` on the currently selected chip). Start/end date inputs (`.dates` grid). Footer: "Reset" ghost link (left) + "Apply" `.btn.sm` button (right). Apply submits the enclosing form. Click-outside dismisses via Bootstrap `data-bs-auto-close="outside"`.

Macro signature:

### `_components/filters_popover.html` — `filters_popover(form_id, groups, button_label="Filters")`

Bootstrap 5 dropdown wrapping chip-group `.pop` content. `groups` is a list of `FilterGroup` objects (or dicts): each has `label` (group label) and `options` (list of `FilterOption` with `label`, `value`, `on: bool`). ## Pydantic View-Models

### `HeroSpec` (new — `app_v2/services/hero_spec.py` or co-located with showcase)

```python
from pydantic import BaseModel
from typing import Literal

class HeroSegment(BaseModel):
    label: str
    value: float       # percentage 0–100
    color: str         # CSS color string e.g. "#3366ff"

class HeroSideStat(BaseModel):
    key: str
    value: str
    tone: Literal["default", "green", "red"] = "default"

class HeroSpec(BaseModel):
    label: str
    big_number: int | float | str
    big_number_unit: str | None = None
    delta_text: str | None = None
    segments: list[HeroSegment] = []
    side_stats: list[HeroSideStat] = []
```

Mirrors `OverviewGridViewModel` / `PageLink` / `ChartSpec` pattern. Type-checked at the router boundary. Showcase constructs a `HeroSpec` instance directly; downstream phases (Platform BM pivot) pass computed instances from their routers.

### `FilterGroup` / `FilterOption` (new — `app_v2/services/filter_spec.py` or inline in showcase)

```python
class FilterOption(BaseModel):
    label: str
    value: str
    on: bool = False

class FilterGroup(BaseModel):
    label: str
    options: list[FilterOption] = []
```

Used by `filters_popover.html` macro. Downstream phases (JV chip-group filter, if added) pass lists of `FilterGroup` from their routers.
