<objective>
Build the domain service layer — the single entry point for all Browse and Settings DB interactions. After this plan, pages no longer touch adapters directly; they call `ufs_service.*` functions that handle caching, server-side filtering, row capping, and result normalization.

Purpose: DATA-05 requires that the full `ufs_data` table is NEVER loaded into pandas — all filtering happens server-side. DATA-06 pivots with `aggfunc="first"` and warns on duplicates. Output: `app/services/ufs_service.py` with the 4-function public API, plus pytest coverage with a SQLite-backed DBAdapter fixture.

**Decision coverage:**
- DATA-05: server-side WHERE IN filters; never full table.
- DATA-06: `pivot_to_wide` uses `aggfunc="first"` and logs warning on duplicates.
- DATA-07: `row_cap=200` (matches `AgentConfig.row_cap`) and a visible `capped` flag returned to callers.
- FOUND-07: `@st.cache_data(ttl=300)` on immutable tuple arguments.
- SAFE-01 (re-asserted at service layer): `SET SESSION TRANSACTION READ ONLY` attempted inside `fetch_cells`.

Pitfall reminders:
- SQLAlchemy 2.x `IN` with tuple: use `bindparam("x", expanding=True)` approach. This is the officially documented SA2 pattern; avoids pymysql tuple-binding ambiguity.

    **fetch_cells(_db, platforms, infocategories, items, row_cap=200)** — decorator `@st.cache_data(ttl=60, show_spinner=False)`. Body:
    ```python
    if not platforms or not items:
        return pd.DataFrame(columns=["PLATFORM_ID", "InfoCategory", "Item", "Result"]), False
    # Empty infocategories is allowed (means "all categories for these items").
    has_cat_filter = bool(infocategories)

    # Build SQL with expanding bindparams (SQLAlchemy 2.x canonical IN clause pattern).
    if has_cat_filter:
        sql = sa.text(
            f"SELECT PLATFORM_ID, InfoCategory, Item, Result FROM {_TABLE} "
            "WHERE PLATFORM_ID IN :platforms "
            "AND InfoCategory IN :categories "
            "AND Item IN :items "
            "LIMIT :cap"
        ).bindparams(
            sa.bindparam("platforms", expanding=True),
            sa.bindparam("categories", expanding=True),
            sa.bindparam("items", expanding=True),
        )
        params = {
            "platforms": list(platforms),
            "categories": list(infocategories),
            "items": list(items),
            "cap": row_cap + 1,
        }
    else:
        sql = sa.text(
            f"SELECT PLATFORM_ID, InfoCategory, Item, Result FROM {_TABLE} "
            "WHERE PLATFORM_ID IN :platforms "
            "AND Item IN :items "
            "LIMIT :cap"
        ).bindparams(
            sa.bindparam("platforms", expanding=True),
            sa.bindparam("items", expanding=True),
        )
        params = {"platforms": list(platforms), "items": list(items), "cap": row_cap + 1}

    with _db._get_engine().connect() as conn:
        # SAFE-01: non-fatal SET SESSION TRANSACTION READ ONLY
        try:
            conn.execute(sa.text("SET SESSION TRANSACTION READ ONLY"))
        except Exception:
            logger.debug("SET SESSION TRANSACTION READ ONLY not supported; continuing")
        df = pd.read_sql_query(sql, conn, params=params)

    capped = len(df) > row_cap
    if capped:
        df = df.head(row_cap)

    # DATA-01/02: normalize Result in place for display (callers may use try_numeric
    # for charts — Stage 5 stays lazy per DATA-02).
    if not df.empty and "Result" in df.columns:
        df["Result"] = normalize(df["Result"])

    return df, capped
    ```

    **pivot_to_wide(df_long, swap_axes=False, col_cap=30)**:
    ```python
    if df_long.empty:
        return df_long.copy(), False

    dup_count = df_long.duplicated(subset=["PLATFORM_ID", "InfoCategory", "Item"]).sum()
    if dup_count > 0:
        logger.warning(
            "pivot_to_wide: %d duplicate (PLATFORM_ID, InfoCategory, Item) rows detected; using aggfunc='first'",
            dup_count,
        )

    if swap_axes:
        index_col, columns_col = "Item", "PLATFORM_ID"
    else:
        index_col, columns_col = "PLATFORM_ID", "Item"

    wide = df_long.pivot_table(
        index=index_col,
        columns=columns_col,
        values="Result",
        aggfunc="first",
    )
    wide.columns.name = None
    wide = wide.reset_index()

    value_cols = [c for c in wide.columns if c != index_col]
    col_capped = False
    if len(value_cols) > col_cap:
        keep = [index_col] + value_cols[:col_cap]
        wide = wide[keep]
        col_capped = True

    return wide, col_capped
    ```

    Constraints:
    - NEVER build SQL with f-string interpolation of user data (platforms, items, categories). Only `_TABLE` is f-string-interpolated — and it is a module constant, not user input.
    - Do NOT call `_db.run_query(sql)` with a formatted SQL string built from user data; use the engine + bindparams path shown.
    - Convert `tuple` args to `list` before passing to expanding bindparams — SQLAlchemy's expanding bindparam accepts `list` natively.
    - The `_db` underscore prefix matters: Streamlit skips hashing of underscore-prefixed args. Cache correctness relies on platforms, infocategories, items, row_cap being the full cache key.
