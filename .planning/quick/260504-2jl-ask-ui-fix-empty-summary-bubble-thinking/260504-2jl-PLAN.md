    display: none;
  }

WHY :has() is safe here:
  - Chrome 105+ (Aug 2022), Safari 15.4+ (Mar 2022), Firefox 121+ (Dec 2023). Universal
    in modern browsers; this is an internal intranet app where the deployed browser is
    known-modern.
  - The predicate scopes specifically to immediate children of .chat-events (`> :not(...)`),
    so deeper descendants don't trigger the match.
  - `:not(.chat-thinking)` includes EVERY SSE event variant — the SSE swap inserts
    elements like .chat-thought, .chat-pill-tool-call, .chat-pill-tool-result-*,
    .chat-text-delta, .chat-final-card, .chat-error-card-*, all of which are NOT
    .chat-thinking → predicate matches → hide.

WHY display: none over .d-none class:
  - No JS path that could fail. The CSS predicate is reactive: it re-evaluates on every
    DOM mutation under .chat-events automatically.
  - .d-none is a Bootstrap utility, but adding it required a working JS event listener;
    the user reported the listener wasn't firing reliably (or was firing too late, or
    on a wrong element).

          }, REVEAL_INTERVAL_MS);
        });
      });
