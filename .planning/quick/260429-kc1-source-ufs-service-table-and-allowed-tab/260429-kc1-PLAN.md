# Quick Task 260429-kc1 — Source ufs_service Table from Settings

## Objective

Two hardcoded module constants in `app/services/ufs_service.py` violated the
single-source-of-truth principle:

```python
_TABLE = "ufs_data"
_ALLOWED_TABLES: frozenset[str] = frozenset({"ufs_data"})
```

Meanwhile, the NL agent already honored `settings.app.agent.allowed_tables`
via `cfg.allowed_tables` — so Browse/Overview queries and the agent could
diverge if a user edited settings.yaml to use a different table.

Make settings.yaml the single source: read `settings.app.agent.allowed_tables`
at module load, derive both the primary table (first entry) and the
allowlist (frozenset of all entries) from it.

## Tasks

### Task 1: Replace hardcoded `_TABLE` / `_ALLOWED_TABLES` with settings reader

**Files:** `app/services/ufs_service.py`

**Action:**

Add `from app.core.config import load_settings` import and a module-level
function:

```python
def _load_table_config() -> tuple[str, frozenset[str]]:
    settings = load_settings()
    allowed = settings.app.agent.allowed_tables
    if not allowed:
        raise RuntimeError(
            "settings.app.agent.allowed_tables is empty — define at least one "
            "table in config/settings.yaml under app.agent.allowed_tables.",
        )
    return allowed[0], frozenset(allowed)


_TABLE, _ALLOWED_TABLES = _load_table_config()
```

Update the module docstring's "Security notes" section to reflect that
`_TABLE` is now a settings-sourced configured constant (not a hardcoded
literal). Keep `_safe_table()` as-is — its semantics are unchanged; only
the allowlist source changed.

**Verify:**

- `grep -F "'ufs_data'" app/services/ufs_service.py` → 0 hits
- `grep -F '_load_table_config' app/services/ufs_service.py` → ≥3 hits (def + call + docstring ref OK)
- `grep -F 'load_settings' app/services/ufs_service.py` → ≥1 hit (import + usage)
- `pytest tests/ -q` exits 0 with no regressions
