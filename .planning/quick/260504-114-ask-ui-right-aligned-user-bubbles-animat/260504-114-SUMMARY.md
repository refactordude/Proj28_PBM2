# Quick Task 260504-114: Ask UI right-aligned user bubbles + animated thinking + streaming text deltas

**One-liner:** Right-aligned user-message bubble (--accent-soft / --radius-card) + CSS-only bouncing-dots Thinking… indicator (with reduced-motion fallback) + new SSE 'text_delta' event that streams the agent's free-text preamble live and hides the indicator on the first event of any kind in the turn.

## Objective recap

Three additive Ask-UI improvements bundled as one quick task:

1. **Right-aligned user bubble + left-aligned assistant prose** — wrap user questions in a right-aligned bubble that uses existing tokens.css colors; assistant content stays left-aligned with no bubble (the asymmetry is the design).
2. **Animated "Thinking…" indicator** — replace the static italic copy with a CSS-only 3-dot bouncing animation; honor `prefers-reduced-motion`.
3. **Stream assistant response** — surface PydanticAI's `TextPart` / `TextPartDelta` events through a new `text_delta` SSE event type; append into a rolling buffer in `.chat-events`; hide the "Thinking…" indicator the moment ANY event of any kind first mounts in that turn's buffer.

Structured-output contract (`PresentResult` via `ToolOutput`) stays intact — the `final` event remains the authoritative source for the final card.

## Tasks executed

### Task 1 — Right-aligned user bubble + animated thinking dots [commit 37dffd0]

**Files:** `app_v2/templates/ask/_user_message.html`, `app_v2/static/css/app.css`

- Replaced the inner markup of `.chat-user-message` to wrap the question in `.chat-user-bubble` and render the thinking row with a 3-span `.chat-thinking-dots` indicator + `.chat-thinking-label`.
- `.chat-user-message` is now a flex column aligned to `flex-end` (right edge of panel-body).
- `.chat-user-bubble` paints the bubble surface: `--accent-soft` fill, `--radius-card` corners with a tightened 6px top-right corner (speech-bubble cue), `max-width: 75%`.
- `.chat-user-question` keeps the prior 15px / 600 / `--ink` typography and adds `white-space: pre-wrap` + `word-break: break-word` so newlines and long unbroken tokens render correctly inside the bubble.
- `.chat-thinking-dots` runs `pbm2-thinking-bounce` 1.4s ease-in-out infinite both with `-0.32s / -0.16s / 0s` per-child stagger (canonical Material/Slack typing-indicator pattern).
- `prefers-reduced-motion` block collapses the animation to `opacity: 1` static dots — they remain visible as a 3-dot indicator with no movement.
- Existing assistant-side styles (`.chat-thought`, `.chat-pill-tool-*`, `.chat-summary-callout`, `.chat-error-card-*`, `.btn-stop`) untouched — the asymmetry is intentional.

### Task 2 — Stream text deltas + hide-thinking-on-first-event [commit c887eb4]

**Files:** `app/core/agent/chat_loop.py`, `app_v2/templates/ask/_text_delta.html` (new), `app_v2/templates/ask/_user_message.html`, `app_v2/templates/ask/index.html`, `app_v2/static/css/app.css`

- Added `TextPartDelta` to the `pydantic_ai.messages` import block; removed `noqa: F401` from `PartDeltaEvent` and `TextPart` (now actively used in `_event_to_payload`).
- Rewrote `_event_to_payload`:
  - `PartStartEvent(TextPart)` with non-empty content → emit `text_delta` (covers providers that ship the first chunk on PartStartEvent rather than on the first PartDeltaEvent).
  - `PartDeltaEvent` whose `delta` is `TextPartDelta` with non-empty `content_delta` → emit `text_delta`.
  - `PartStartEvent(ToolCallPart)` → still skipped (the FunctionToolCallEvent below carries the visible pill).
  - All other branches (`ThinkingPart`, `FunctionToolCallEvent`, `FunctionToolResultEvent`) unchanged.
- Added `_render_text_delta(content)` helper that renders the new `_text_delta.html` partial via `templates.get_template(...).render(delta=content)`.
- New `_text_delta.html`: a single `<span class="chat-text-delta">{{ delta | e }}</span>`, autoescaped (no `| safe`), with the `pre-wrap` + `--ink` styling defined in CSS so consecutive deltas concatenate visually as one paragraph inside the rolling buffer.
- Extended `_user_message.html`'s `.chat-events` `sse-swap` attribute to include `text_delta` (allow-list addition; no other event handling touched).
- Added a second IIFE inside `{% block scripts %}` of `index.html` guarded by `window.__pbm2HideThinkingBound`. On `htmx:sseMessage`, walks back from the event-target's closest `.chat-events` through preceding siblings to find the matching `.chat-user-message` for that turn, then adds Bootstrap `.d-none` to its `.chat-thinking` if not already present (idempotent).
- Added `.chat-text-delta` CSS rule: `white-space: pre-wrap; word-break: break-word; font-size: 14px; color: var(--ink); line-height: 1.6;`.
