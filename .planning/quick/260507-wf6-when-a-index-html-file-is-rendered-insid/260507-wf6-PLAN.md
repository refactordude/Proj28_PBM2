<objective>
Two atomic UI fixes on the Joint Validation detail page:

1. **Iframe gutter:** Remove the `p-0` Bootstrap utility from the iframe's wrapper so `.panel-body`'s default `padding: 26px 32px` (lines 58 of app.css) restores horizontal breathing room around the embedded `index.html`. Currently the wrapper is `<div class="panel-body p-0">`, which strips ALL padding and lets the iframe sit flush against the rounded panel edges — visually too tight.

2. **Sticky topbar:** Add `position: sticky; top: 12px; z-index: 50` to the existing `.topbar` rule in app.css. Today the topbar scrolls offscreen the moment the user scrolls down on the iframe page (because the iframe is `height: 80vh` and the user has to scroll past it to see the rest of the page), making the `Joint Validation` tab effectively unreachable without scrolling all the way back to the top. Sticky-positioning keeps it pinned to the viewport so the tab is always one click away.

Purpose: Restore visual breathing room around the Confluence iframe; eliminate the "tab effectively broken" UX where the topbar scrolls offscreen.

Output: Two-file, ~6-line diff. No new tokens, no JS, no new test fixtures.
</objective>


**Constraints:**
- Add the three new declarations INSIDE the existing `.topbar { ... }` block — do NOT create a second `.topbar` rule. Two `.topbar { ... }` blocks would double the test_phase04_uif_invariants.py:36 `.topbar {` substring count, but the existing test only asserts presence, not count, so a duplicate would technically pass. Still: keep it as ONE rule to avoid dead-code drift.
- The Jinja comment is a CSS comment (`/* ... */`), not a Jinja `{# ... #}` comment.
- Do NOT touch any other rule in the file. Specifically, do NOT touch:
  - `.tabs`, `.tab`, `.tab[aria-selected="true"]`, `.tab .count` rules (lines ~828-859).
  - `.brand`, `.brand-mark`, `.av`, `.top-right` rules.
  - The `.panel { overflow: hidden }` rule (line 56) — overflow:hidden on the parent of a sticky child does NOT clip stickiness because `.topbar` is a child of `<body>`, not of any `.panel`.
  - The `body { display: flex; flex-direction: column; min-height: 100vh; }` rule (lines 9-13). Sticky elements work inside flex containers as long as the flex container has scrollable overflow — `<body>` does.

**Why these specific values:**
- `position: sticky` (NOT `fixed`): keeps the topbar in document flow at scroll-top (no jump / layout-shift), and pins it only when the viewport scrolls past its initial position.
- `top: 12px`: matches the existing `margin-top: 12px` so the pill visually re-settles into the same spot it occupied at scroll-top — no perceived jank.
- `z-index: 50`: above panels (which have no z-index), below Bootstrap dropdown/modal layers (typically 1000+). Avoids colliding with the `.table-sticky-corner` z-index ladder (1/2/3) which is in its own table-scoped stacking context anyway.
