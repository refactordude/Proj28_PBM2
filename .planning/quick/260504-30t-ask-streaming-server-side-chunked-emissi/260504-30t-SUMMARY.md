# Quick task 260504-30t: Server-side chunked summary streaming as text_delta SSE events Summary

**One-liner:** Server-side chunker in `chat_loop.stream_chat_turn` splits the agent's `PresentResult.summary` into word-level chunks and emits one `text_delta` SSE event per chunk with ~20ms between chunks, BEFORE yielding the `final` event — making streaming actually visible in the chat surface while preserving the structured-output contract and reusing all existing 260504-114 plumbing.

## What changed

### `app/core/agent/chat_loop.py` (+52 / -1 lines net)

1. **Import:** added `import re` (alphabetical between `logging` and `typing`).
2. **Module constant:** added `_STREAM_CHUNK_DELAY_MS = 20` near `_THOUGHT_TRUNCATE_CAP` with a 7-line comment explaining the trade-off.
3. **Chunker block:** in `stream_chat_turn`, between the `AgentRunResultEvent` capture and the `yield {"event": "final", ...}` line, inserted:
   - 18-line doc-comment explaining the contract (why server-side chunking, why cancel-during-stream is graceful and not an error).
   - `summary = output.summary or ""` + `if summary.strip():` guard so empty/whitespace-only summaries skip chunking.
   - `chunks = re.findall(r"\S+\s*", summary)` — preserves trailing whitespace.
   - `for i, chunk in enumerate(chunks):` with cancel check at TOP, yield `text_delta`, sleep BETWEEN chunks via `if i < len(chunks) - 1: await asyncio.sleep(_STREAM_CHUNK_DELAY_MS / 1000)`.
4. **Final yield byte-stable:** the existing `final_payload = {summary, sql, chart_spec_dict, new_messages}` dict and `yield {"event": "final", ...}` line preserved verbatim — router contract unchanged.

### `app_v2/templates/ask/_final_card.html` (+26 / -10 lines net)

1. **Removed the `.chat-summary-callout` block** (lines 38-46 in the prior version) entirely — the summary is now painted by streamed `text_delta` events that mount into `.chat-events` before the final card.
2. **Updated doc-comment block** to:
   - Add a 260504-30t entry in the iterative-history list (alongside 260504-220 and 260504-2jl).
   - Drop the `{{ summary | e }}` line from the AUTOESCAPE CONTRACT.
   - Replace `Three vertical sections` with `Two vertical sections` (table + chart).
   - Rephrase prior 260504-2jl reference to retire the `chat-summary-callout` literal so the migration-completeness grep returns 0.
3. **Inline replacement comment** at the prior summary-block site explains the removal and points to `chat_loop.stream_chat_turn / _STREAM_CHUNK_DELAY_MS` for the chunking cadence.
4. **Byte-stable:** `chat-final-card` root div, table block (`{% if table_html %} ... {% endif %}`), chart block (`{% if chart_html %} ... {% endif %}`), and OOB `<div id="input-zone" hx-swap-oob="true">` swap all preserved verbatim.

### `app_v2/static/css/app.css` (untouched)

Per Task 1.C decision: the existing `.chat-final-card { margin-top: 12px }` rule (line 577-579) provides 12px top margin between the streamed inline `.chat-text-delta` spans (which terminate the inline flow above) and the block-level `.chat-final-card` div below — adequate separation. The orphaned `.chat-summary-callout` CSS rule (line 559-567) left in place as harmless dead code per the constraints. If checkpoint verification reveals the rhythm is too tight, a one-character bump (12 → 16px) is the planned remediation.
