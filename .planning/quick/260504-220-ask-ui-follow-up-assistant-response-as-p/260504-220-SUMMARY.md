# Quick Task 260504-220: Ask UI follow-up — assistant response as prose Summary

Three fixes to the Ask chat surface following user feedback on quick task
260504-114: drop the .panel/.panel-body chrome from the assistant's final
response; move the "Thinking…" indicator from under the right-aligned user
bubble to the left-aligned assistant column; perceive the agent's response
as "alive" via word-by-word reveal of the summary text once the final card
mounts.

## What Shipped

### Task 1 — Markup + CSS restructure (commit `a738e3e`)

**`_user_message.html`:**
- `.chat-thinking` block relocated from inside `.chat-user-message` (right-aligned)
  to be the FIRST child of `.chat-events` (left-aligned, assistant column)
- User bubble byte-stable from quick task 260504-114

**`_final_card.html`:**
- Dropped `.panel/.panel-body` wrappers — root is now plain `<div class="chat-final-card">`
- Summary text wrapped per-word in `<span class="sw">` spans
- `.chat-summary-callout` background (var(--accent-soft)) preserved as the only surviving visual emphasis
- Table + chart sections unchanged; OOB `#input-zone` swap unchanged

**`index.html`:**
- Hide-thinking handler simplified — `eventsDiv.querySelector(".chat-thinking")` directly, no `previousElementSibling` walk-back

**`app.css`:**
- `.chat-final-card` chrome-stripped to `margin-top: 12px` only (no background, border, shadow, or padding)

- New `.chat-summary-callout .sw.is-visible { opacity: 1 }` rule
- New `prefers-reduced-motion` override forcing all `.sw` spans visible at once

### Task 2 — Word-reveal IIFE (commit `90d2530`)

**`index.html`:**
- Added `__pbm2RevealSummaryBound`-guarded IIFE inside `{% block scripts %}`
- Listens for `htmx:sseMessage` filtered on `evt.detail.type === "final"`
- `requestAnimationFrame` defers walk one frame so htmx finishes inserting the final-card markup
- Targets the LAST `.chat-summary-callout` (most recently mounted)
- `setInterval` at 30ms cadence adds `.is-visible` to each `.sw` span in document order
- A typical 1-2 sentence summary completes reveal in ~600-1200ms

## Concern 3 root-cause confirmation

The structured-output contract (`agent = Agent(..., output_type=ToolOutput(PresentResult, name="present_result"))`)
forces PydanticAI providers (OpenAI + Ollama) to deliver the model's response
through the `present_result` tool-call channel as `ToolCallPartDelta` events.
Standalone `TextPart` / `TextPartDelta` events are not emitted by the providers
when a tool-call is forced/expected. The `text_delta` SSE plumbing from quick
task 260504-114 is correct (verified at `app_v2/templates/ask/_text_delta.html`
and `app/core/agent/chat_loop._event_to_payload`) but the events never arrive
in practice.

The lightest correct fix is therefore the client-side word-reveal on the
final summary: zero protocol change, identical behavior across both LLM
backends, no race with htmx swap timing, and the existing `text_delta`
plumbing remains as additive future-headroom (zero cost; will automatically
benefit any future model that does emit free text preambles).

## Cross-backend results

The word-reveal animation is fully client-side — backend choice has zero
effect on it. Both OpenAI and Ollama terminate via the `present_result`
tool-call (per the structured-output contract); both deliver the final
payload to the same `_hydrate_final_card` render path; both trigger the
same `htmx:sseMessage` of type `final`; both run through the same
`__pbm2RevealSummaryBound` IIFE. Behavior is identical by construction.
